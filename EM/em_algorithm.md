## Background: The Latent Variable Problem

### What is a latent variable model, and why does it cause trouble?

In many real-world problems, the data we observe is generated by a process that involves
quantities we cannot directly measure. These unobserved quantities are called **latent
variables** — "latent" here means *hidden* or *unobserved*, not small or insignificant.
They are genuinely part of the generative process; we just cannot see them.

Three canonical examples:

- **Gaussian mixture model**: we observe heights of people, but not their gender. Gender is the latent variable driving which Gaussian each height was drawn from.
- **Hidden Markov model**: we observe audio waveforms, but not the phoneme sequence being spoken. The phonemes are the latent variable.
- **Topic model**: we observe word frequencies in documents, but not the topic distribution. Topics are latent.

In all three cases, the generative story is:

1. Draw a latent variable $Z$ according to some distribution $p(Z; \theta)$
2. Draw the observed variable $X$ from the conditional $p(X | Z; \theta)$

The joint distribution over both is $p(X, Z; \theta)$, and the **marginal distribution** over what we actually observe — the quantity we can evaluate on data — is:

$$p(X; \theta) = \sum_{Z} p(X, Z; \theta) \qquad \text{(or an integral if } Z \text{ is continuous)}$$

This marginalisation — summing over all possible values of $Z$ — is where the trouble starts.

### Why direct maximum likelihood fails

Given $n$ observations $x_1, \ldots, x_n$, we want to find parameters $\theta$ that maximise the
log-likelihood:

$$\ell(\theta) = \sum_{i=1}^n \log p(x_i; \theta) = \sum_{i=1}^n \log \underbrace{\sum_{Z} p(x_i, Z; \theta)}_{\text{sum over all } Z \text{ values}}$$

The outer sum over data points is fine — each term $\log p(x_i; \theta)$ is independent. The
problem is the **log of a sum** inside each term. To see why this causes trouble, apply the
chain rule to differentiate $\ell(\theta)$ with respect to a single parameter. By the chain
rule applied to $\log(\cdot)$:

$$\frac{\partial \ell}{\partial \theta} = \sum_{i=1}^n \frac{1}{p(x_i;\theta)} \cdot \frac{\partial}{\partial \theta} p(x_i;\theta) = \sum_{i=1}^n \frac{1}{\sum_Z p(x_i, Z; \theta)} \cdot \sum_Z \frac{\partial}{\partial \theta} p(x_i, Z; \theta)$$

where in the second step we substituted $p(x_i;\theta) = \sum_Z p(x_i, Z; \theta)$ and pushed
the derivative inside the sum over $Z$ (valid by linearity of differentiation). Setting this to
zero does not yield a clean equation for $\theta$ because the denominator $\sum_Z p(x_i, Z;
\theta)$ depends on $\theta$ in the same complicated way as the numerator — you are left with
$\theta$ entangled on both sides of the equation simultaneously.

To see this concretely, take a Gaussian mixture model with $K$ components. Differentiating
$\ell(\theta)$ with respect to the mean $\mu_k$ of component $k$ and setting to zero eventually
gives:

$$\mu_k = \frac{\sum_{i=1}^n r_{ik} \, x_i}{\sum_{i=1}^n r_{ik}}, \qquad \text{where} \qquad r_{ik} = \frac{\pi_k \,\mathcal{N}(x_i; \mu_k, \Sigma_k)}{\sum_{k'} \pi_{k'}\,\mathcal{N}(x_i; \mu_{k'}, \Sigma_{k'})}$$

This looks like a solution for $\mu_k$ — a weighted average of the data — but the weights
$r_{ik}$ themselves depend on $\mu_k$ through the Gaussian density in the numerator and
denominator. So the "solution" for $\mu_k$ requires knowing $\mu_k$ to compute the weights.
There is no way to isolate $\mu_k$ algebraically; the system has no closed form. This is the
**circular coupling** that makes direct MLE intractable for mixture models.

*A side note on the $K^n$ argument*: you may sometimes see the claim that mixture model MLE
is intractable because there are $K^n$ possible latent assignments to enumerate. This is
actually only true for models where the latents are *coupled across data points* — such as
HMMs, where the sequence of hidden states must be inferred jointly. For a GMM, the latent
$z_i$ for each observation $x_i$ is independent of all other $z_j$, so the likelihood
factorises as $p(\mathbf{x};\theta) = \prod_i \sum_k \pi_k \mathcal{N}(x_i;\mu_k,\Sigma_k)$ and
is computable in $O(nK)$ time. The GMM obstruction is purely the circular coupling above,
not combinatorial explosion.

### The key insight: if we knew Z, everything would be easy

Suppose, hypothetically, that we were given the latent variables alongside the observations — the so-called **complete data** $(x_1, z_1), \ldots, (x_n, z_n)$. The complete-data log-likelihood is:

$$\ell_{\text{complete}}(\theta) = \sum_{i=1}^n \log p(x_i, z_i; \theta) = \sum_{i=1}^n \left[ \log p(z_i; \theta) + \log p(x_i | z_i; \theta) \right]$$

This is a **sum of logs** — no log of a sum. For exponential family models (Gaussians,
categorical distributions, etc.), this typically has a closed-form maximum. The problem is
that we do not know $z_i$.

The EM algorithm resolves this tension: instead of hard-assigning each $z_i$ to a single
value (which would require committing to something we don't know), it works with the
**expected complete-data log-likelihood** — a soft, probabilistic version that replaces each
unknown $z_i$ with a distribution over all its possible values, weighted by how plausible
each value is given the current parameters.

## Part 1: The ELBO — Constructing a Tractable Lower Bound

### The core strategy

Rather than maximising $\ell(\theta) = \sum_i \log p(x_i; \theta)$ directly, EM constructs a **lower
bound** to $\ell(\theta)$ that is (a) easy to maximise and (b) guaranteed to touch $\ell(\theta)$ at the
current parameter estimate. Maximising this bound gives a new parameter estimate that
cannot have decreased $\ell(\theta)$.

The bound is built using **Jensen's inequality**, so we begin there.

### Jensen's inequality

**Definition**: A function $f$ is **concave** if for any two points $a, b$ in its domain and any $\lambda \in [0,1]$:
$$f(\lambda a + (1-\lambda) b) \geq \lambda f(a) + (1-\lambda)f(b)$$

Geometrically: the function lies *above* the chord connecting any two of its points. The
archetypal concave function is $\log$.

**Jensen's inequality** generalises this to expectations: if $f$ is concave and $X$ is any
random variable, then:

$$f\!\left(\mathbb{E}[X]\right) \geq \mathbb{E}[f(X)]$$

In words: the function of the average is at least as large as the average of the function.
With strict concavity (like $\log$), equality holds if and only if $X$ is constant with
probability 1.

Concretely with $f = \log$, a discrete distribution $q$, and any positive function $r(Z)$
(a placeholder for whatever non-negative quantity sits inside the sum — we will set $r(Z) = p(x,Z;\theta)$ momentarily):

$$\log \sum_Z q(Z) \cdot \frac{r(Z)}{q(Z)} \geq \sum_Z q(Z) \cdot \log \frac{r(Z)}{q(Z)}$$

The left side is $\log \mathbb{E}_{Z \sim q}[r(Z)/q(Z)]$ and the right side is $\mathbb{E}_{Z \sim q}[\log(r(Z)/q(Z))]$. This is exactly the manipulation we will use below.

### Deriving the Evidence Lower BOund (ELBO)

Fix a single observation $x$ for now (we will sum over all data points at the end). We want
to lower-bound $\log p(x; \theta)$.

**Step 1: Introduce an arbitrary distribution $q(Z)$.**

Let $q(Z)$ be any probability distribution over the latent variable $Z$ — we will
optimise our choice of $q$ later, but for now it is completely free. We use $q$ to
multiply and divide inside the log without changing anything:

$$\log p(x; \theta) = \log \sum_Z p(x, Z; \theta)$$

Multiply and divide each term inside the sum by $q(Z)$:

$$= \log \sum_Z q(Z) \cdot \frac{p(x, Z; \theta)}{q(Z)}$$

**Step 2: Apply Jensen's inequality.**

Recognise the expression as $\log \mathbb{E}_{Z \sim q}\!\left[\frac{p(x, Z; \theta)}{q(Z)}\right]$.
Since $\log$ is concave, Jensen's inequality gives:

$$\log \mathbb{E}_{Z \sim q}\!\left[\frac{p(x, Z; \theta)}{q(Z)}\right] \geq \mathbb{E}_{Z \sim q}\!\left[\log \frac{p(x, Z; \theta)}{q(Z)}\right]$$

**Step 3: Expand the expectation.**

$$\mathbb{E}_{Z \sim q}\!\left[\log \frac{p(x, Z; \theta)}{q(Z)}\right] = \underbrace{\sum_Z q(Z) \log p(x, Z; \theta)}_{\mathbb{E}_q[\log p(x,Z;\theta)]} + \underbrace{\left(-\sum_Z q(Z) \log q(Z)\right)}_{H(q)}$$

where $H(q) = -\sum_Z q(Z) \log q(Z)$ is the **entropy** of $q$ — a non-negative scalar
measuring how spread out $q$ is. Uniform $q$ has maximum entropy; a distribution that puts
all mass on one value has entropy zero.

**Step 4: Name the result.**

Combining steps 2 and 3, we have:

$$\boxed{\log p(x; \theta) \geq \underbrace{\mathbb{E}_{Z \sim q}\!\left[\log p(x, Z; \theta)\right]}_{\text{expected complete log-likelihood}} + \underbrace{H(q)}_{\text{entropy of }q} \;\triangleq\; \mathcal{L}(\theta, q)}$$

The right-hand side is called the **Evidence Lower BOund** or **ELBO**, written $\mathcal{L}(\theta, q)$.

*A note on $q(Z)$*: at this point $q$ is still a completely free, arbitrary distribution over $Z$ — we have not said what it should be. You might wonder how $q(Z)$ relates to $p(Z;\theta)$, which already defines a distribution over $Z$ via the model parameters. They play different roles and the relationship will become precise in Part 2. For now, the key point is that the bound holds *for any* choice of $q$, which is what makes it general.

Reading term by term:
- $\mathbb{E}_{Z \sim q}[\log p(x, Z; \theta)]$: how well the model explains the complete data on average, where "on average" is taken over the distribution $q$ on the hidden variable $Z$. We want this large.
- $H(q)$: the entropy of our distribution $q$ over $Z$. Recall that entropy is zero for a point mass (fully committed to one value) and maximised for a uniform distribution (fully spread out). In classical EM, $q$ is set to the exact posterior $p(Z|x;\theta_t)$ and this term is simply whatever it happens to be — it is not tuned separately. The entropy term only plays an active role in settings like VAEs (covered in Part 6), where $q$ is an approximate posterior that we are free to push around and might otherwise collapse to a degenerate point mass.

The key achievement: we have converted a **log of a sum** (hard) into a **sum of logs** (tractable), at the cost of introducing a lower bound.

### The gap: ELBO vs. the true log-likelihood

How loose is this bound? Start by expanding $\mathcal{L}(\theta, q)$ using its definition and the fact that $H(q) = -\mathbb{E}_q[\log q(Z)]$:

$$\mathcal{L}(\theta, q) = \mathbb{E}_q[\log p(x, Z; \theta)] + H(q) = \mathbb{E}_q[\log p(x, Z; \theta)] - \mathbb{E}_q[\log q(Z)]$$

The gap between the true log-likelihood and the ELBO is therefore:

$$\log p(x; \theta) - \mathcal{L}(\theta, q) = \log p(x; \theta) - \mathbb{E}_q[\log p(x, Z; \theta)] + \mathbb{E}_q[\log q(Z)]$$

We will now show this equals the **KL divergence** from $q(Z)$ to the true posterior $p(Z|x;\theta)$.

The **KL divergence** from distribution $q$ to distribution $p$ is:

$$\text{KL}(q \| p) = \sum_Z q(Z) \log \frac{q(Z)}{p(Z)} \geq 0$$

The non-negativity follows from Jensen's inequality applied to $-\log$ (which is convex). Since $-\log$ is convex, Jensen gives $-\log \mathbb{E}_q[p/q] \leq \mathbb{E}_q[-\log(p/q)]$. The key step is evaluating the left side: $\mathbb{E}_q[p(Z)/q(Z)] = \sum_Z q(Z) \cdot p(Z)/q(Z) = \sum_Z p(Z) = 1$, so the left side is $-\log(1) = 0$. The right side is $\mathbb{E}_q[\log(q/p)] = \text{KL}(q\|p)$. Therefore $0 \leq \text{KL}(q\|p)$. KL equals zero if and only if $p(Z)/q(Z)$ is constant with probability 1 under $q$, which (since both integrate to 1) means $q = p$ everywhere.

Now compute the gap:

$$\log p(x; \theta) - \mathcal{L}(\theta, q)$$

$$= \log p(x; \theta) - \sum_Z q(Z) \log p(x, Z; \theta) + \sum_Z q(Z) \log q(Z)$$

Use $p(x, Z; \theta) = p(Z | x; \theta) \cdot p(x; \theta)$ to write $\log p(x, Z; \theta) = \log p(Z|x;\theta) + \log p(x;\theta)$:

$$= \log p(x;\theta) - \sum_Z q(Z)\left[\log p(Z|x;\theta) + \log p(x;\theta)\right] + \sum_Z q(Z) \log q(Z)$$

Since $\sum_Z q(Z) = 1$, the $\log p(x;\theta)$ terms cancel:

$$= -\sum_Z q(Z) \log p(Z|x;\theta) + \sum_Z q(Z) \log q(Z) = \sum_Z q(Z) \log \frac{q(Z)}{p(Z|x;\theta)} = \text{KL}(q \| p(\cdot|x;\theta))$$

So:

$$\boxed{\log p(x; \theta) = \mathcal{L}(\theta, q) + \text{KL}\!\left(q(Z) \;\|\; p(Z|x;\theta)\right)}$$

This decomposition is central to everything that follows. Reading it:
- The true log-likelihood = ELBO + a non-negative gap
- The ELBO is always a lower bound, and is tight when the gap is zero
- The gap is zero if and only if $q(Z) = p(Z|x;\theta)$ — i.e., when $q$ equals the true posterior

**So why did we do all this work just to end up back at $\log p(x;\theta)$?**

Because we have rewritten a single intractable quantity as the sum of two tractable handles. Before this decomposition, $\log p(x;\theta)$ was a monolithic object — a log of a sum that we couldn't optimise directly. Now we can see exactly what it's made of: an ELBO term we *can* compute and maximise, plus a KL gap that tells us how much we're leaving on the table.

The whole point of EM is to maximise $\log p(x;\theta)$ — to find the parameters that make the observed data as probable as possible. Since we can't do this directly, we use the ELBO as a proxy: a lower bound we *can* optimise. The ELBO depends on two things — the parameters $\theta$ and the auxiliary distribution $q$ — and EM optimises them one at a time, alternating.

**Step 1 — E-step: fix $\theta_t$, optimise $q$.**
Set $q = p(Z|x;\theta_t)$. This makes the KL gap zero, so the ELBO equals the true log-likelihood at the current parameters: $\mathcal{L}(\theta_t, q^*) = \log p(x;\theta_t)$. We now have a lower bound that is exact right here.

**Step 2 — M-step: fix $q^*$, optimise $\theta$.**
Find the $\theta$ that maximises $\mathcal{L}(\theta, q^*)$. Call it $\theta_{t+1}$. Since we moved to a higher point on the ELBO, and the ELBO is always a lower bound on the true log-likelihood, the true log-likelihood at $\theta_{t+1}$ must be at least as high as the ELBO there — which is at least as high as where we started. So $\log p(x;\theta_{t+1}) \geq \log p(x;\theta_t)$.

**Step 3 — Repeat.**
At the new $\theta_{t+1}$, the old $q^*$ is stale — it was computed from $\theta_t$, not $\theta_{t+1}$. So the KL gap has reopened and the ELBO is no longer exact. Go back to Step 1 with $\theta_{t+1}$ and repeat.

This is exactly the EM algorithm. Step 1 is the E-step. Step 2 is the M-step. The decomposition is not just a mathematical identity — it is the blueprint for the algorithm.

## Part 2: The E-Step — Finding the Tightest Bound

### What the E-step does

At iteration $t$, we have a current parameter estimate $\theta_t$. The E-step selects the
distribution $q$ that makes the ELBO as tight as possible at $\theta_t$. From the
decomposition above, we want to minimize the KL gap, which means setting:

$$q^*(Z) = p(Z \mid x; \theta_t)$$

With this choice, $\text{KL}(q^* \| p(\cdot|x;\theta_t)) = 0$, and the ELBO touches the true
log-likelihood at the current estimate:

$$\mathcal{L}(\theta_t, q^*) = \log p(x; \theta_t)$$

This is the **tightest possible lower bound** at $\theta_t$.

### Formal derivation via Lagrange multipliers

We can also derive $q^* = p(Z|x;\theta_t)$ directly, without the KL argument, by
maximising $\mathcal{L}(\theta_t, q)$ over the distribution $q$ subject to $\sum_Z q(Z) = 1$.

The ELBO as a function of $q$ (holding $\theta_t$ fixed) is:

$$\mathcal{L}(\theta_t, q) = \sum_Z q(Z) \log p(x, Z; \theta_t) - \sum_Z q(Z) \log q(Z)$$

We maximise this over $q$ subject to $\sum_Z q(Z) = 1$. Introduce a Lagrange multiplier $\lambda$ to enforce the constraint. The Lagrangian is:

$$G(q) = \lambda\!\left[1 - \sum_Z q(Z)\right] + \sum_Z q(Z) \log p(x, Z; \theta_t) - \sum_Z q(Z) \log q(Z)$$

Taking the derivative with respect to $q(Z)$ for each specific value $Z$
(treating $q(Z)$ as a free variable for each $Z$ independently) and setting it to zero.
The $\lambda$ and $\log p$ terms differentiate straightforwardly. The entropy term
$-q(Z)\log q(Z)$ requires the product rule:

$$\frac{d}{dq(Z)}\left[-q(Z)\log q(Z)\right] = -\log q(Z) - q(Z)\cdot\frac{1}{q(Z)} = -\log q(Z) - 1$$

Combining all three terms:

$$\frac{\partial G}{\partial q(Z)} = -\lambda + \log p(x, Z; \theta_t) - \log q(Z) - 1 = 0$$

Solving for $q(Z)$:

$$\log q(Z) = \log p(x, Z; \theta_t) - \lambda - 1$$

$$q(Z) = \frac{p(x, Z; \theta_t)}{e^{\lambda + 1}}$$

The denominator $e^{\lambda+1}$ is a constant (does not depend on $Z$). Apply the normalisation
constraint $\sum_Z q(Z) = 1$ to determine it:

$$\sum_Z q(Z) = \frac{\sum_Z p(x, Z; \theta_t)}{e^{\lambda+1}} = \frac{p(x;\theta_t)}{e^{\lambda+1}} = 1 \implies e^{\lambda+1} = p(x;\theta_t)$$

Substituting back:

$$q^*(Z) = \frac{p(x, Z; \theta_t)}{p(x;\theta_t)} = p(Z \mid x; \theta_t) \checkmark$$

The E-step sets $q$ equal to the posterior over latent variables given the observations and
the current parameters. This is not a choice — it is the uniquely optimal distribution.

**Why this matters.** Recall from Part 1 that the gap between the ELBO and the true log-likelihood is exactly $\text{KL}(q \| p(\cdot|x;\theta_t))$. The only way to reduce that gap to zero is to set $q$ equal to the posterior — no other distribution can do it. This means the E-step is not an arbitrary heuristic; it is the mathematically inevitable answer to the question "what $q$ makes the bound as tight as possible?"

### What the posterior computes in practice

In concrete models, computing $p(Z|x;\theta_t)$ means using Bayes' theorem:

$$p(Z = k \mid x; \theta_t) = \frac{p(x \mid Z=k; \theta_t) \cdot p(Z=k; \theta_t)}{\sum_{k'} p(x \mid Z=k'; \theta_t) \cdot p(Z=k'; \theta_t)}$$

Reading this: the numerator scores how well component $k$ explains the observation $x$,
weighted by how probable component $k$ is a priori. The denominator normalises so the
probabilities sum to 1 over all components. These are called **responsibilities** — how
much credit component $k$ takes for having generated observation $x$.

## Part 3: The M-Step — Maximising the Bound

### What the M-step does

After the E-step, we have a tight lower bound $\mathcal{L}(\theta, q^*)$ where $q^*(Z) = p(Z|x;\theta_t)$
is fixed. The M-step maximises $\mathcal{L}(\theta, q^*)$ over the free variable $\theta$:

$$\theta_{t+1} = \arg\max_\theta \; \mathcal{L}(\theta, q^*)$$

**Simplification**: Since $H(q^*)$ does not depend on $\theta$ (it is a constant once $q^*$ is
fixed in the E-step), maximising $\mathcal{L}(\theta, q^*)$ is equivalent to maximising just the
first term:

$$\theta_{t+1} = \arg\max_\theta \; \mathbb{E}_{Z \sim q^*}\!\left[\log p(x, Z; \theta)\right]$$

Substituting $q^*(Z) = p(Z|x;\theta_t)$:

$$\theta_{t+1} = \arg\max_\theta \sum_Z p(Z \mid x; \theta_t) \log p(x, Z; \theta) \;\triangleq\; \arg\max_\theta \; Q(\theta \mid \theta_t)$$

The function $Q(\theta \mid \theta_t)$ is the **expected complete-data log-likelihood**:
$$\boxed{Q(\theta \mid \theta_t) = \mathbb{E}_{Z \mid x,\, \theta_t}\!\left[\log p(x, Z; \theta)\right]}$$

Reading this expression:
- $\log p(x, Z; \theta)$ is the complete-data log-likelihood — the log-likelihood we *would* have if we knew $Z$
- The expectation $\mathbb{E}_{Z|x,\theta_t}[\cdot]$ replaces the unknown $Z$ with a soft weighted average over all its values, where the weights are the posterior probabilities from the E-step
- The parameter $\theta$ inside the log is the **free variable** we are optimising — this is what changes from iteration to iteration
- The parameter $\theta_t$ in the expectation is the **fixed** current estimate used to compute the weights — it does not move during the M-step

The intuition: rather than filling in $Z$ with a single hard guess (which would require
knowing the right answer), the M-step fills it in *softly*, giving each possible value of $Z$
a weight equal to its posterior probability. This is sometimes called **soft EM** to
distinguish it from **hard EM**, which replaces the responsibility distribution with a point
mass — assigning each observation $x_i$ entirely to whichever component $k$ maximises
$r_{ik}$. Hard EM is related to $k$-means, but the equivalence is exact only in the special
case of a GMM where all components have identical spherical covariances $\Sigma_k = \sigma^2 I$.
In that case the Gaussian likelihoods reduce to Euclidean distances, hard assignment selects
the nearest centroid, and the M-step computes plain unweighted means — exactly $k$-means
clustering. For general GMMs with different covariances, the assignment step uses the full
Gaussian likelihood (which implicitly measures Mahalanobis distance), but the M-step still
just computes ordinary unweighted means and covariances over all hard-assigned points — it
is not $k$-means, but the M-step itself is not doing anything Mahalanobis-weighted either.

For exponential family models, $Q(\theta | \theta_t)$ is typically a concave function of $\theta$ with
a closed-form maximiser — exactly the tractability promised at the outset.

**We now have a complete algorithm.** The E-step gives us a tractable way to handle the unknown latents (replace them with a soft distribution). The M-step gives us a tractable way to update the parameters (maximise a surrogate objective with a closed form). The two steps together form one iteration of EM. What we have not yet shown is that iterating them is guaranteed to make progress — that is the job of Part 4.

## Part 4: Convergence Guarantee — Two Proofs

### Why we need a proof

EM is a local procedure: each iteration only looks at the current parameter value and
takes one step. There is no global view of the likelihood surface. Before trusting the
algorithm, we need to prove that each step cannot *decrease* the likelihood — otherwise
we would have no guarantee that anything useful is happening.

We will prove: $L(\theta_{t+1}) \geq L(\theta_t)$, where $L(\theta) = \log p(x;\theta)$.

The two proofs attack this from different angles, and both are worth understanding.

---

### Proof 1: Via KL Divergence (Wasserman's approach)

This proof works at the level of the raw log-likelihood and traces exactly which
algebraic steps cause the increase.

**Setup.** Define the function $J$ as (following Wasserman's notation, adapted):

$$J(\theta \mid \theta_t) = \mathbb{E}_{\theta_t}\!\left[\log \frac{p(x, Z; \theta)}{p(x, Z; \theta_t)} \;\Bigg|\; x\right]$$

where the expectation is over $Z$ drawn from the posterior $p(Z|x;\theta_t)$.

Note that $J(\theta_t | \theta_t) = \mathbb{E}[\log 1 \mid x] = 0$ — the ratio equals 1 when $\theta = \theta_t$,
so its log is zero. Before continuing, we should verify that the M-step — which was defined in
§3 as maximising $Q(\theta|\theta_t)$ — is equivalent to maximising $J(\theta|\theta_t)$. Expanding:

$$J(\theta \mid \theta_t) = \mathbb{E}_{\theta_t}\!\left[\log p(x, Z; \theta) \mid x\right] - \mathbb{E}_{\theta_t}\!\left[\log p(x, Z; \theta_t) \mid x\right] = Q(\theta \mid \theta_t) - Q(\theta_t \mid \theta_t)$$

The second term $Q(\theta_t | \theta_t)$ is a constant with respect to $\theta$, so maximising $J$
over $\theta$ is identical to maximising $Q$ over $\theta$. Therefore, the M-step maximising $Q$ does
indeed also maximise $J$, and we have $J(\theta_{t+1}|\theta_t) \geq J(\theta_t|\theta_t) = 0$.

**Step 1: Factorise $J$ using the chain rule of probability.**

Write $p(x, Z; \theta) = p(x;\theta) \cdot p(Z|x;\theta)$:

$$J(\theta \mid \theta_t) = \mathbb{E}_{\theta_t}\!\left[\log \frac{p(x;\theta) \cdot p(Z|x;\theta)}{p(x;\theta_t) \cdot p(Z|x;\theta_t)} \;\Bigg|\; x\right]$$

Split the log of the product into two terms:

$$= \mathbb{E}_{\theta_t}\!\left[\log \frac{p(x;\theta)}{p(x;\theta_t)} \;\Bigg|\; x\right] + \mathbb{E}_{\theta_t}\!\left[\log \frac{p(Z|x;\theta)}{p(Z|x;\theta_t)} \;\Bigg|\; x\right]$$

**Step 2: Simplify the first term.**

The first term does not depend on $Z$ — it is a constant inside the expectation:

$$\mathbb{E}_{\theta_t}\!\left[\log \frac{p(x;\theta)}{p(x;\theta_t)} \;\Bigg|\; x\right] = \log \frac{p(x;\theta)}{p(x;\theta_t)} = L(\theta) - L(\theta_t)$$

**Step 3: Identify the second term as a negative KL divergence.**

$$\mathbb{E}_{\theta_t}\!\left[\log \frac{p(Z|x;\theta)}{p(Z|x;\theta_t)} \;\Bigg|\; x\right] = \sum_Z p(Z|x;\theta_t) \log \frac{p(Z|x;\theta)}{p(Z|x;\theta_t)} = -\text{KL}\!\left(p(\cdot|x;\theta_t) \;\|\; p(\cdot|x;\theta)\right)$$

This is the negative KL divergence from the old posterior to the new posterior — it is $\leq 0$.

**Step 4: Rearrange.**

Combining steps 1–3:

$$J(\theta \mid \theta_t) = \left[L(\theta) - L(\theta_t)\right] - \text{KL}\!\left(p(\cdot|x;\theta_t) \;\|\; p(\cdot|x;\theta)\right)$$

Solving for the likelihood change:

$$L(\theta) - L(\theta_t) = J(\theta \mid \theta_t) + \text{KL}\!\left(p(\cdot|x;\theta_t) \;\|\; p(\cdot|x;\theta)\right)$$

**Step 5: Conclude.**

Evaluating at $\theta = \theta_{t+1}$:

$$L(\theta_{t+1}) - L(\theta_t) = \underbrace{J(\theta_{t+1} \mid \theta_t)}_{\geq 0, \text{ by M-step}} + \underbrace{\text{KL}\!\left(p(\cdot|x;\theta_t) \;\|\; p(\cdot|x;\theta_{t+1})\right)}_{\geq 0, \text{ always}} \geq 0 \checkmark$$

Both terms are non-negative: the first because the M-step chose $\theta_{t+1}$ to maximise $J$
(and $J(\theta_t|\theta_t) = 0$ is one feasible value), and the second because KL divergence is always non-negative. Therefore $L(\theta_{t+1}) \geq L(\theta_t)$.

---

### Proof 2: Via the ELBO (Dellaert's approach)

This proof works geometrically and is often more intuitive. It tracks what happens to the
ELBO across the two steps.

**Setup.** Recall the fundamental decomposition from Part 1:

$$L(\theta) = \log p(x;\theta) = \mathcal{L}(\theta, q) + \text{KL}(q \| p(\cdot|x;\theta))$$

This holds for *any* $q$ and *any* $\theta$.

**After the E-step.** The E-step sets $q_t = p(Z|x;\theta_t)$. With this choice:

$$\text{KL}(q_t \| p(\cdot|x;\theta_t)) = 0$$

So the ELBO equals the log-likelihood at the current estimate:

$$\mathcal{L}(\theta_t, q_t) = L(\theta_t)$$

Geometrically: the lower bound surface $\mathcal{L}(\theta, q_t)$ (as a function of $\theta$) *touches*
the log-likelihood surface $L(\theta)$ at the single point $\theta = \theta_t$, and lies strictly
below it everywhere else. To see why, recall the decomposition:

$$L(\theta) = \mathcal{L}(\theta, q_t) + \text{KL}(q_t \| p(\cdot|x;\theta))$$

At $\theta = \theta_t$, the KL gap is zero because $q_t$ was specifically constructed to equal
$p(\cdot|x;\theta_t)$. But at any other $\theta \neq \theta_t$, the posterior $p(\cdot|x;\theta)$ has
a different shape — the $q_t$ we fixed in the E-step will generally not match it — so the KL
gap is strictly positive and the ELBO lies strictly below the true likelihood there. This is
why each E-step constructs a *fresh* lower bound that is tight at the current $\theta$ but
not at any old $\theta$.

**After the M-step.** The M-step finds $\theta_{t+1}$ that maximises $\mathcal{L}(\theta, q_t)$ over $\theta$,
while $q_t$ stays fixed. Therefore:

$$\mathcal{L}(\theta_{t+1}, q_t) \geq \mathcal{L}(\theta_t, q_t) = L(\theta_t)$$

Now evaluate the fundamental decomposition at $(\theta_{t+1}, q_t)$:

$$L(\theta_{t+1}) = \mathcal{L}(\theta_{t+1}, q_t) + \underbrace{\text{KL}(q_t \| p(\cdot|x;\theta_{t+1}))}_{\geq 0}$$

Therefore:

$$L(\theta_{t+1}) \geq \mathcal{L}(\theta_{t+1}, q_t) \geq \mathcal{L}(\theta_t, q_t) = L(\theta_t) \checkmark$$

**What each proof illuminates.** The KL proof shows the mechanism precisely: the
guarantee comes from two separate non-negative quantities (the M-step gain and the KL
between old and new posteriors). The ELBO proof shows the geometric picture: the E-step
lifts the bound up to touch the likelihood; the M-step slides along the bound surface to a
higher point; the next E-step re-lifts. Each cycle ratchets the likelihood upward.

[FIG:ORIGINAL — EM convergence geometry, two panels: (left) the true log-likelihood curve L(θ) as a smooth function of θ, with successive ELBO lower-bound curves L(θ, q_t) drawn as parabola-like curves each touching L(θ) from below at exactly θ_t; label the M-step as a horizontal arrow along the bound from θ_t to θ_{t+1}, and the E-step as a vertical lift at θ_{t+1} from the old bound up to L(θ_{t+1}); show 2–3 iterations to illustrate the ratcheting pattern; (right) a multi-modal L(θ) showing two different initialisations converging to different local maxima] The
algorithm can stop improving only when $\theta_t$ is a stationary point of $L(\theta)$ — a
local maximum, saddle point, or in degenerate cases a local minimum — at which point the
M-step finds no better value and the likelihood is unchanged.

**A critical caveat.** The guarantee is monotone increase, *not* convergence to the global
maximum. The likelihood surface for latent variable models typically has multiple local
maxima. EM is guaranteed to climb to *some* local maximum (or saddle point), but which
one depends on initialisation. Multiple random restarts are standard practice.

**Rate of convergence.** Both proofs guarantee monotone increase but say nothing about *speed*. In practice EM can require hundreds or thousands of iterations to converge, especially near the solution. The theoretical reason is that EM has **linear convergence** with a rate determined by the *fraction of missing information*:

$$\text{rate} = 1 - \frac{I_\text{obs}(\theta)}{I_\text{comp}(\theta)}$$

where $I_\text{obs}(\theta)$ is the observed Fisher information (curvature of $\ell(\theta)$) and $I_\text{comp}(\theta)$ is the complete-data Fisher information (curvature of $\ell_c(\theta)$, which would be available if we knew $Z$). The ratio $I_\text{obs}/I_\text{comp} \in (0, 1]$ is the *fraction of observed information* — how much of the "total" statistical information is accessible from the observed data alone. When the latent variables carry most of the information (low $I_\text{obs}/I_\text{comp}$), the convergence rate approaches 1 and EM takes very many steps. Intuitively: if you are very uncertain about the latent assignments, each EM step can only nudge the parameters a little, because the soft responsibilities change only slightly with each parameter update.

**Practical implication:** For GMMs with well-separated clusters, convergence is fast (the assignments are nearly hard, so little information is hidden). For GMMs with heavily overlapping clusters, convergence is slow — exactly when you most need many iterations. Common accelerations include ECME (which replaces some M-step substeps with direct maximisation of $\ell(\theta)$), quasi-Newton M-steps, and variance-reduction techniques. For modern deep models, gradient-based M-steps (GEM) are standard and often converge faster due to momentum and adaptive step sizes.

### Generalised EM: the M-step doesn't need to be exact

Looking back at both convergence proofs, notice what they actually required of the M-step. Proof 1 needed $J(\theta_{t+1}|\theta_t) \geq J(\theta_t|\theta_t) = 0$ — that is, $\theta_{t+1}$ must *increase* $J$ above zero. Proof 2 needed $\mathcal{L}(\theta_{t+1}, q_t) \geq \mathcal{L}(\theta_t, q_t)$ — that is, $\theta_{t+1}$ must *increase* the ELBO. Neither proof required $\theta_{t+1}$ to be the *maximum* of $Q$. The proofs go through as long as the M-step makes any upward progress — it does not need to find the best possible step.

This observation defines **Generalised EM (GEM)**: replace the exact M-step maximisation with any update that satisfies $Q(\theta_{t+1}|\theta_t) \geq Q(\theta_t|\theta_t)$. A single gradient ascent step on $Q$, for instance, satisfies this whenever the step size is small enough. The monotone likelihood guarantee is preserved exactly.

**Why this matters.** Classical EM requires a closed-form $\arg\max$ of $Q(\theta|\theta_t)$, which exists for exponential family models but fails the moment the model is too complex (e.g., a neural network decoder). GEM removes this requirement — any M-step that improves $Q$ is sufficient for the convergence guarantee. This is the theoretical justification for gradient-based EM variants and, more broadly, for the many modern algorithms that perform partial M-steps via gradient descent rather than exact optimisation.

**A related relaxation: MCEM.** Sometimes $Q(\theta|\theta_t) = \mathbb{E}_{Z|x,\theta_t}[\log p(x,Z;\theta)]$ itself cannot be computed analytically because the posterior expectation has no closed form. **Monte Carlo EM (MCEM)** approximates $Q$ by drawing samples $z^{(1)}, \ldots, z^{(M)}$ from the current posterior $p(Z|x;\theta_t)$ and replacing the expectation with a sample average:

$$Q(\theta|\theta_t) \approx \frac{1}{M} \sum_{m=1}^M \log p(x, z^{(m)}; \theta)$$

This approximation improves as $M$ grows, and the resulting update can be treated as a GEM step. MCEM is rarely needed for clean models like GMMs where $Q$ is analytic, but becomes essential when the posterior expectation resists closed form.

## Part 5: Worked Example — Gaussian Mixture Model

### Motivation: why a mixture model?

A Gaussian mixture model (GMM) is the most natural setting for EM because it makes
the latent variable structure completely explicit, the E-step posterior has a clean closed
form, and the M-step reduces to weighted means and covariances that are straightforwardly
derived from calculus.

It is also the companion example referenced in both readings, so we derive everything in full.

### Setup

We have $n$ observations $x_1, \ldots, x_n \in \mathbb{R}^d$ drawn from a mixture of $K$ Gaussian
components. The generative model is:

1. Draw a component assignment: $z_i \sim \text{Categorical}(\pi_1, \ldots, \pi_K)$, where $\pi_k \geq 0$ and $\sum_k \pi_k = 1$
2. Draw the observation: $x_i \mid z_i = k \sim \mathcal{N}(\mu_k, \Sigma_k)$

The parameters are $\theta = \{\pi_k, \mu_k, \Sigma_k\}_{k=1}^K$.

The **observed likelihood** for a single point is the marginal over $z_i$:

$$p(x_i; \theta) = \sum_{k=1}^K \pi_k \, \mathcal{N}(x_i; \mu_k, \Sigma_k)$$

The **observed log-likelihood** over all data is:

$$\ell(\theta) = \sum_{i=1}^n \log \sum_{k=1}^K \pi_k \, \mathcal{N}(x_i; \mu_k, \Sigma_k)$$

This is the log of a sum — analytically intractable, as established in the Background section.

The **complete-data log-likelihood** (if we knew the $z_i$) is:

$$\ell_c(\theta) = \sum_{i=1}^n \log p(x_i, z_i; \theta) = \sum_{i=1}^n \left[\log \pi_{z_i} + \log \mathcal{N}(x_i; \mu_{z_i}, \Sigma_{z_i})\right]$$

This factors cleanly: the mixing weight term $\log \pi_{z_i}$ depends only on $\pi$, and the
Gaussian term depends only on $\mu_{z_i}, \Sigma_{z_i}$.

### The $Q$ function

The complete-data log-likelihood for a single observation $x_i$, given that $z_i$ takes one
specific value, can be written using an **indicator function** $\mathbf{1}[z_i = k]$ — which
equals 1 when $z_i = k$ and 0 otherwise — to select the relevant component:

$$\log p(x_i, z_i; \theta) = \sum_{k=1}^K \mathbf{1}[z_i = k]\left[\log \pi_k + \log \mathcal{N}(x_i; \mu_k, \Sigma_k)\right]$$

This is just a notational device: since $z_i$ equals exactly one value, all terms in the sum
are zero except the one where $k = z_i$, which recovers $\log \pi_{z_i} + \log
\mathcal{N}(x_i;\mu_{z_i},\Sigma_{z_i})$. We write it this way because it makes taking expectations over $z_i$ mechanical. Recall that the M-step objective is $Q(\theta\mid\theta_t) = \mathbb{E}_{z_i\mid x_i,\theta_t}[\log p(x_i, z_i;\theta)]$ — the expectation of the complete-data log-likelihood under the current posterior. So we take the expectation of both sides under $p(z_i\mid x_i;\theta_t)$. On the right side, the terms $\log \pi_k$ and $\log \mathcal{N}(x_i;\mu_k,\Sigma_k)$ do not depend on $z_i$ — they are constants inside the expectation — so the expectation passes through to land only on the indicator. We then use the standard fact that the expectation of an indicator function equals the probability of the event it indicates:

$$\mathbb{E}_{z_i \mid x_i,\theta_t}\!\left[\mathbf{1}[z_i = k]\right] = p(z_i = k \mid x_i; \theta_t) = r_{ik}$$

This is not a trick — it is a direct consequence of what $Q$ is defined to be. The replacement converts the hard binary indicator (1 if $z_i = k$, 0 otherwise) into a soft fractional weight $r_{ik} \in [0,1]$, which is precisely the move from hard to soft assignment.

Summing over all data points and taking the expectation gives the $Q$ function:

$$Q(\theta \mid \theta_t) = \sum_{i=1}^n \sum_{k=1}^K \underbrace{p(z_i = k \mid x_i; \theta_t)}_{r_{ik}} \left[\log \pi_k + \log \mathcal{N}(x_i; \mu_k, \Sigma_k)\right]$$

where we define the **responsibility** $r_{ik} \triangleq p(z_i = k \mid x_i; \theta_t)$ — the posterior
probability that component $k$ generated observation $i$ under the current parameters.

**This is the tractability payoff.** The Background section identified the core obstruction as a log of a sum: $\log \sum_k \pi_k \mathcal{N}(x_i;\mu_k,\Sigma_k)$. The $Q$ function has no such term — the log now sits *inside* the sum, applied directly to $\log \pi_k$ and $\log \mathcal{N}$. The unknown discrete latent $z_i$ has been replaced by soft weights $r_{ik}$, and the result is a weighted sum of logs that we can differentiate and maximise in closed form. The indicator function trick was the mechanism; this is why it mattered.

### The E-Step: Computing Responsibilities

By Bayes' theorem, the responsibility of component $k$ for observation $i$ is:

$$r_{ik} = p(z_i = k \mid x_i; \theta_t) = \frac{p(x_i \mid z_i = k; \theta_t) \cdot p(z_i = k; \theta_t)}{\sum_{k'=1}^K p(x_i \mid z_i = k'; \theta_t) \cdot p(z_i = k'; \theta_t)}$$

Substituting the model:

$$\boxed{r_{ik} = \frac{\pi_k^t \, \mathcal{N}(x_i;\, \mu_k^t,\, \Sigma_k^t)}{\displaystyle\sum_{k'=1}^K \pi_{k'}^t \, \mathcal{N}(x_i;\, \mu_{k'}^t,\, \Sigma_{k'}^t)}}$$

Reading left to right: the numerator asks "how well does component $k$ explain observation
$i$?" — scoring both by the prior weight $\pi_k^t$ and the Gaussian likelihood. The
denominator normalises so responsibilities for observation $i$ sum to 1 across all components.

Define the **effective count** for component $k$:

$$N_k = \sum_{i=1}^n r_{ik}$$

This is not an integer but a soft count — the expected number of observations assigned to
component $k$. We have $\sum_k N_k = n$, which follows by swapping the order of summation
and using the fact that responsibilities sum to 1 over components for each observation:

$$\sum_k N_k = \sum_k \sum_{i=1}^n r_{ik} = \sum_{i=1}^n \sum_k r_{ik} = \sum_{i=1}^n 1 = n$$

### The M-Step: Deriving All Parameter Updates

We now maximise $Q(\theta | \theta_t)$ over $\theta = (\pi, \mu_1, \ldots, \mu_K, \Sigma_1, \ldots, \Sigma_K)$.
To see why the blocks can be optimised independently, expand $Q$ by splitting the log of the joint into its two additive parts:

$$Q(\theta \mid \theta_t) = \underbrace{\sum_{i=1}^n \sum_{k=1}^K r_{ik} \log \pi_k}_{Q_\pi\text{: depends only on }\pi} + \underbrace{\sum_{i=1}^n \sum_{k=1}^K r_{ik} \log \mathcal{N}(x_i; \mu_k, \Sigma_k)}_{Q_\mu,\Sigma\text{: depends only on }\mu_k, \Sigma_k}$$

Because the $r_{ik}$ are constants during the M-step (fixed by the E-step), the two groups share no parameters. Maximising $Q$ over all parameters simultaneously is therefore equivalent to maximising each group separately — a standard result for sums of functions of disjoint variables.

#### Updating the mixing weights $\pi$

Isolate the $\pi$-dependent terms from $Q$:

$$Q_\pi = \sum_{i=1}^n \sum_{k=1}^K r_{ik} \log \pi_k$$

We maximise this subject to $\sum_k \pi_k = 1$, $\pi_k \geq 0$. The inequality constraint can be handled separately: since the responsibilities $r_{ik} \geq 0$ for all $i, k$ (they are probabilities), the effective count $N_k = \sum_i r_{ik} \geq 0$, which means the unconstrained optimum $\pi_k = N_k/n$ will automatically satisfy $\pi_k \geq 0$. So we only need a Lagrange multiplier for the equality constraint. Introduce $\lambda$ for the equality $\sum_k \pi_k = 1$. The Lagrangian is:

$$G(\pi) = \sum_{i=1}^n \sum_{k=1}^K r_{ik} \log \pi_k + \lambda\!\left(1 - \sum_k \pi_k\right)$$

Take the derivative with respect to $\pi_k$ (treating each $\pi_k$ independently) and set to zero:

$$\frac{\partial G}{\partial \pi_k} = \sum_{i=1}^n \frac{r_{ik}}{\pi_k} - \lambda = 0$$

Solving: $\pi_k = \frac{1}{\lambda} \sum_{i=1}^n r_{ik} = \frac{N_k}{\lambda}$.

Apply the normalisation constraint $\sum_k \pi_k = 1$ to find $\lambda$:

$$\sum_k \pi_k = \frac{1}{\lambda} \sum_k N_k = \frac{n}{\lambda} = 1 \implies \lambda = n$$

Substituting back:

$$\boxed{\pi_k^{t+1} = \frac{N_k}{n} = \frac{\displaystyle\sum_{i=1}^n r_{ik}}{n}}$$

This is the empirical mixture weight: the fraction of observations (in expectation) assigned
to component $k$. If component $k$ is responsible for 30 out of 100 observations on
average, its weight becomes 0.30.

#### Updating the means $\mu_k$

Isolate the $\mu_k$-dependent terms from $Q$ (the Gaussian log-density for component $k$):

$$Q_{\mu_k} = \sum_{i=1}^n r_{ik} \log \mathcal{N}(x_i; \mu_k, \Sigma_k)$$

Expand the Gaussian log-density:

$$\log \mathcal{N}(x_i; \mu_k, \Sigma_k) = -\frac{d}{2}\log(2\pi) - \frac{1}{2}\log|\Sigma_k| - \frac{1}{2}(x_i - \mu_k)^T \Sigma_k^{-1}(x_i - \mu_k)$$

Only the last term depends on $\mu_k$, so:

$$Q_{\mu_k} = \text{const} - \frac{1}{2} \sum_{i=1}^n r_{ik} \,(x_i - \mu_k)^T \Sigma_k^{-1}(x_i - \mu_k)$$

Take the gradient with respect to $\mu_k$. The standard identity for a quadratic form is
$\nabla_a(a^T M a) = 2Ma$ for symmetric $M$. Here $a = x_i - \mu_k$, and we want the
gradient with respect to $\mu_k$, not $a$. By the chain rule:

$$\frac{\partial}{\partial \mu_k}(x_i - \mu_k)^T \Sigma_k^{-1}(x_i - \mu_k) = \frac{\partial a}{\partial \mu_k} \cdot \nabla_a(a^T \Sigma_k^{-1} a) = (-I) \cdot 2\Sigma_k^{-1}(x_i - \mu_k) = -2\Sigma_k^{-1}(x_i - \mu_k)$$

Applying this to $Q_{\mu_k} = \text{const} - \frac{1}{2}\sum_i r_{ik}(x_i-\mu_k)^T\Sigma_k^{-1}(x_i-\mu_k)$, and noting that the $-\frac{1}{2}$ outside cancels with the factor of $-2$ from the gradient:

$$\nabla_{\mu_k} Q_{\mu_k} = -\frac{1}{2}\sum_{i=1}^n r_{ik} \cdot \left(-2\Sigma_k^{-1}(x_i - \mu_k)\right) = \sum_{i=1}^n r_{ik} \,\Sigma_k^{-1}(x_i - \mu_k) = 0$$

Since $\Sigma_k^{-1}$ is invertible (positive definite), we can multiply both sides on the left by
$\Sigma_k$ to get:

$$\sum_{i=1}^n r_{ik}(x_i - \mu_k) = 0$$

$$\sum_{i=1}^n r_{ik} x_i = \mu_k \sum_{i=1}^n r_{ik} = \mu_k N_k$$

$$\boxed{\mu_k^{t+1} = \frac{\displaystyle\sum_{i=1}^n r_{ik}\, x_i}{N_k}}$$

This is the **responsibility-weighted mean** — a weighted average of all observations,
where each observation $x_i$ contributes in proportion to how much credit component $k$
takes for having generated it. An observation near the centre of component $k$'s current
Gaussian gets high weight; a far-away observation gets low weight.

#### Updating the covariances $\Sigma_k$

It is algebraically convenient to differentiate with respect to the **precision matrix**
$\Lambda_k = \Sigma_k^{-1}$ (the inverse of the covariance). The reason is not that the
log-density is linear in $\Lambda_k$ — it is not — but that both $\Lambda_k$-dependent terms
in the log-density have clean, known matrix derivative formulas, which we derive now.

Isolate the $\Lambda_k$-dependent terms in the Gaussian log-density (absorbing constants not
involving $\Lambda_k$):

$$Q_{\Lambda_k} = \frac{1}{2}\sum_{i=1}^n r_{ik} \left[\log|\Lambda_k| - (x_i - \mu_k)^T \Lambda_k (x_i - \mu_k)\right] + \text{const}$$

We need two matrix derivative identities. We derive both.

**Identity 1**: $\dfrac{\partial}{\partial \Lambda} \log|\Lambda| = \Lambda^{-T}$.

For a symmetric positive definite matrix, $\Lambda^{-T} = \Lambda^{-1} = \Sigma$. The derivation
uses the formula for how a determinant changes with a small perturbation: for a positive
definite $\Lambda$, $|\Lambda + \epsilon E| \approx |\Lambda|(1 + \epsilon \,\text{tr}(\Lambda^{-1}E))$
for small $\epsilon$ and any matrix $E$. Taking the log: $\log|\Lambda + \epsilon E| \approx
\log|\Lambda| + \epsilon\,\text{tr}(\Lambda^{-1}E)$. The directional derivative in direction $E$ is
therefore $\text{tr}(\Lambda^{-1}E) = \text{tr}(\Sigma E)$. Since this equals $\text{tr}(\Sigma^T E)$
(as $\Sigma$ is symmetric), the gradient is $\Sigma = \Lambda^{-1}$.

**Identity 2**: $\dfrac{\partial}{\partial \Lambda} \left[a^T \Lambda a\right] = aa^T$ for a fixed vector $a$.

Write the quadratic form as a trace. First note that $a^T \Lambda a$ is a scalar, so
$a^T \Lambda a = \text{tr}(a^T \Lambda a)$. Applying the cyclic property of trace
($\text{tr}(ABC) = \text{tr}(CAB)$) to cycle $a^T$ to the back:
$\text{tr}(a^T \cdot \Lambda a) = \text{tr}(\Lambda a \cdot a^T) = \text{tr}(\Lambda \, aa^T) = \text{tr}(aa^T \Lambda)$.

Now differentiate. Since $\text{tr}(aa^T \Lambda) = \sum_{m,n}(aa^T)_{mn}\Lambda_{nm}$,
differentiating with respect to $\Lambda_{ij}$ picks out the $(j,i)$ entry of $aa^T$,
i.e. $(aa^T)_{ji}$. The matrix of all such derivatives is therefore $(aa^T)^T = aa^T$
(since $aa^T$ is symmetric), giving the identity.

Taking the gradient of $Q_{\Lambda_k}$ and applying both identities term by term:

$$\frac{\partial Q_{\Lambda_k}}{\partial \Lambda_k} = \frac{1}{2}\sum_{i=1}^n r_{ik}\left[\frac{\partial \log|\Lambda_k|}{\partial \Lambda_k} - \frac{\partial}{\partial \Lambda_k}(x_i - \mu_k)^T \Lambda_k (x_i - \mu_k)\right]$$

Substituting Identity 1 ($\partial \log|\Lambda_k|/\partial\Lambda_k = \Sigma_k$) into the first term, and Identity 2 ($\partial[a^T\Lambda_k a]/\partial\Lambda_k = aa^T$ with $a = x_i - \mu_k$) into the second:

$$= \frac{1}{2}\sum_{i=1}^n r_{ik}\left[\Sigma_k - (x_i - \mu_k)(x_i - \mu_k)^T\right] = 0$$

Multiplying through by $2$:

$$N_k \Sigma_k = \sum_{i=1}^n r_{ik}(x_i - \mu_k)(x_i - \mu_k)^T$$

$$\boxed{\Sigma_k^{t+1} = \frac{\displaystyle\sum_{i=1}^n r_{ik}\,(x_i - \mu_k^{t+1})(x_i - \mu_k^{t+1})^T}{N_k}}$$

**Note on $\mu_k^{t+1}$ vs. $\mu_k^t$.** The derivation above treated $\mu_k$ as a fixed constant while differentiating with respect to $\Lambda_k$. Strictly, this means the formula is valid for any fixed value of $\mu_k$. In practice we substitute the freshly-updated mean $\mu_k^{t+1}$ — computed earlier in the same M-step — rather than the stale $\mu_k^t$. This substitution is valid and, importantly, gives a *better* (lower) covariance estimate: using the updated mean reduces the residuals $x_i - \mu_k$ on average, so the resulting $\Sigma_k^{t+1}$ more accurately captures the spread around the current centroid. Substituting the old $\mu_k^t$ would be formally correct but would overstate the variance by the distance $\|\mu_k^{t+1} - \mu_k^t\|^2$ (by the bias-variance decomposition of the second moment).

This is the **responsibility-weighted sample covariance** — a soft version of the standard
covariance estimator. Each outer product $(x_i - \mu_k)(x_i - \mu_k)^T$ measures how
observation $i$ deviates from the component mean, as a $d \times d$ matrix capturing spread
in every direction and cross-direction. Weighting by $r_{ik}$ gives more influence to
observations that most plausibly belong to component $k$.

**The three updates form a self-consistent system.** The means drift toward the data points each component is responsible for. The covariances adapt to capture the spread of those same points around the updated means. The mixing weights reflect each component's share of the total dataset. None of these updates can be computed without the responsibilities $r_{ik}$ from the E-step — and the responsibilities themselves depend on the current $(\pi_k, \mu_k, \Sigma_k)$. This mutual dependence is exactly the circular coupling that made direct MLE impossible; EM breaks the circle by alternating between fixing the responsibilities and fixing the parameters.

### High-level summary of one EM iteration for a GMM

| Step | What is computed | How |
|---|---|---|
| **E-step** | $r_{ik}$ for all $i, k$ | Bayes' theorem: score each component on each point, normalise |
| **M-step: $\pi$** | New mixing weights | Fraction of total expected counts: $N_k / n$ |
| **M-step: $\mu$** | New component means | Responsibility-weighted centroid of all observations |
| **M-step: $\Sigma$** | New component covariances | Responsibility-weighted sample covariance around new means |

The algorithm iterates E and M steps until the observed log-likelihood $\ell(\theta)$ stops
increasing meaningfully (measured by a tolerance $|\ell(\theta_{t+1}) - \ell(\theta_t)| < \epsilon$).

### A Critical Failure Mode: Degenerate Covariances

There is a fundamental pathology of GMM-EM that every practitioner must know about. Consider what happens when a component $k$ collapses onto a single data point $x_j$: the mean drifts toward $x_j$ and the covariance shrinks to near-zero. As $\Sigma_k \to 0$, the Gaussian density $\mathcal{N}(x_j; \mu_k, \Sigma_k) \to \infty$ — the component becomes an infinitely sharp spike at $x_j$, assigning it responsibility $r_{kj} \to 1$ and all others $r_{ki} \to 0$. The contribution of component $k$ to the observed log-likelihood is then $\log(\pi_k \mathcal{N}(x_j;\mu_k,\Sigma_k)) \to +\infty$.

This means the **observed log-likelihood is unbounded above**. There is no finite global maximum of $\ell(\theta)$ — the likelihood surface has genuine singularities at degenerate covariances, and EM will race toward them if a component happens to collapse. This is not a local optimum; it is a non-solution that makes the model useless (a single component becomes a lookup table for one point, with no generalisation).

**Why it happens.** Collapse is most likely when (a) $k$ is too large (more components than real clusters), (b) one cluster is initialised very close to a single data point, or (c) early assignments give a component only one or two points.

**Standard fixes:**
- **Covariance regularisation:** add a small floor $\Sigma_k \leftarrow \Sigma_k + \epsilon I$ after each M-step update, preventing eigenvalues from reaching zero. $\epsilon$ is typically set to a small fraction of the data's overall variance.
- **Minimum eigenvalue constraint:** after each update, clip any eigenvalue of $\Sigma_k$ below a threshold $\lambda_{\min}$.
- **Bayesian / MAP estimation:** place an inverse-Wishart prior on $\Sigma_k$. The MAP M-step adds a prior-dependent offset to the covariance, effectively regularising automatically. This is the principled fix used by MCLUST and related software.
- **Restart on collapse:** detect a collapsed component (when $N_k < d + 1$, there are too few effective counts to estimate a $d \times d$ covariance) and reinitialise that component randomly.

**So what?** This failure mode explains why naive GMM-EM implementations are unreliable — not because the algorithm is wrong, but because the model's likelihood surface is ill-posed without constraints. Any production implementation must guard against covariance collapse.

## Part 6: Connections to Modern ML

### Why this matters

EM is not a historical curiosity. Its core idea — that optimising over a latent variable model
by alternating between "infer the latents" and "update the parameters" — is the
structural ancestor of a large family of modern algorithms. The ELBO, in particular,
appears under different names throughout modern deep learning.

### Connection 1: The ELBO in Variational Autoencoders (VAEs)

#### The problem EM cannot solve

EM requires computing the exact posterior $p(Z|x;\theta)$ in the E-step. For models with
chain structure (HMMs), the **forward-backward algorithm** — a dynamic programming procedure
that exploits the Markov chain structure to pass messages forward and backward through the
sequence, computing exact posterior marginals over hidden states in $O(nK^2)$ time — handles
this exactly. For a GMM, the latent variable $z_i$ is discrete with only $K$ values, so the posterior
$p(z_i=k|x_i;\theta)$ is simply a $K$-way Bayes' rule — directly evaluable by computing $K$
Gaussian densities and normalising. But for a deep generative model with a neural network
decoder, the posterior over latent variables $z$ given an observation $x$ is:

$$p_\theta(z|x) = \frac{p_\theta(x|z) \cdot p(z)}{p_\theta(x)} = \frac{p_\theta(x|z) \cdot p(z)}{\int p_\theta(x|z) p(z) \, dz}$$

The denominator $p_\theta(x) = \int p_\theta(x|z) p(z) \, dz$ is intractable — it requires
integrating a neural network over all of latent space. The E-step cannot be performed exactly.

#### The VAE fix: amortised variational inference

The VAE replaces the exact posterior with a **parameterised approximate posterior**
(the **encoder**):

$$q_\phi(z|x) \approx p_\theta(z|x)$$

where $q_\phi$ is a neural network with parameters $\phi$ (e.g., one that outputs the mean
and diagonal covariance of a Gaussian). This is called **amortised inference** because
the network learns to produce approximate posteriors for *any* input $x$, rather than
computing a separate posterior for each $x$ from scratch.

#### Deriving the VAE ELBO from scratch

Start with the same inequality we derived in Part 1, but now with continuous $z$ and the parameterised approximate posterior $q_\phi(z|x)$:

$$\log p_\theta(x) = \log \int p_\theta(x, z) \, dz = \log \int q_\phi(z|x) \cdot \frac{p_\theta(x, z)}{q_\phi(z|x)} \, dz$$

Applying Jensen's inequality (the log is concave, so we move it inside):

$$\geq \int q_\phi(z|x) \cdot \log \frac{p_\theta(x, z)}{q_\phi(z|x)} \, dz = \mathbb{E}_{z \sim q_\phi(z|x)}\!\left[\log \frac{p_\theta(x, z)}{q_\phi(z|x)}\right]$$

Expand using $p_\theta(x, z) = p_\theta(x|z) \cdot p(z)$:

$$= \mathbb{E}_{z \sim q_\phi(z|x)}\!\left[\log p_\theta(x|z) + \log \frac{p(z)}{q_\phi(z|x)}\right]$$

The second term is $\mathbb{E}_{q_\phi}[\log p(z)/q_\phi(z|x)] = -\mathbb{E}_{q_\phi}[\log q_\phi(z|x)/p(z)]$.
Recognising the definition of KL divergence — $\text{KL}(q\|p) = \mathbb{E}_q[\log q/p]$ — this equals
$-\text{KL}(q_\phi(z|x)\|p(z))$. So the full expression becomes:

$$= \underbrace{\mathbb{E}_{z \sim q_\phi(z|x)}\!\left[\log p_\theta(x|z)\right]}_{\text{reconstruction term}} - \underbrace{\text{KL}\!\left(q_\phi(z|x) \;\|\; p(z)\right)}_{\text{regularisation term}}$$

This is the **VAE ELBO**:

$$\boxed{\mathcal{L}_\text{VAE}(\theta, \phi) = \underbrace{\mathbb{E}_{z \sim q_\phi(z|x)}\!\left[\log p_\theta(x|z)\right]}_{\text{reconstruction}} - \underbrace{\text{KL}\!\left(q_\phi(z|x) \;\|\; p(z)\right)}_{\text{regularisation}}}$$

Reading each term:

- **Reconstruction term**: $\mathbb{E}_{z \sim q_\phi(z|x)}[\log p_\theta(x|z)]$ — sample a latent code $z$ from the encoder $q_\phi(z|x)$, then score how well the decoder $p_\theta(x|z)$ reconstructs the original input. Large means the decoder is explaining the data well.
- **Regularisation (KL) term**: $\text{KL}(q_\phi(z|x) \| p(z))$ — penalises the approximate posterior for diverging from the prior $p(z) = \mathcal{N}(0, I)$. This prevents the encoder from collapsing to a deterministic mapping (which would make $z$ uninformative) and encourages the latent space to be smooth and well-structured.

**Why this matters.** Classical EM is stuck the moment the E-step becomes intractable — for a neural network decoder, computing the true posterior $p_\theta(z|x)$ requires integrating over all of latent space, which is impossible. The VAE sidesteps this entirely by replacing the E-step with a trained neural network $q_\phi$ that *learns* to approximate the posterior from data. Both $\theta$ and $\phi$ are then optimised jointly via gradient descent on the ELBO. This is what makes deep generative modelling at scale possible — the same ELBO framework, but with intractable exact inference replaced by learned approximate inference.

The gap between the true log-likelihood and the ELBO is $\text{KL}(q_\phi(z|x) \| p_\theta(z|x))$ — 
exactly as in classical EM. Minimising this gap (bringing the approximate posterior closer
to the true posterior) is the role of the encoder.

#### Comparing EM and VAE

| Concept | Classical EM | VAE |
|---|---|---|
| Posterior | Computed exactly: $p(Z|x;\theta)$ | Approximated by encoder: $q_\phi(z|x)$ |
| E-step | Set $q = $ exact posterior (zero KL gap) | Update $\phi$ to reduce KL gap |
| M-step | Maximise $Q(\theta|\theta_t)$ in closed form | Update $\theta$ via gradient descent |
| Objective | $\log p(x;\theta)$ | VAE ELBO $\mathcal{L}_\text{VAE}(\theta, \phi)$ |
| Optimisation | Coordinate ascent (alternating) | Joint gradient descent on $(\theta, \phi)$ |

The structural identity is complete: the VAE ELBO is the same object as the EM ELBO,
with an approximate posterior substituted for the exact one, and gradient descent
substituted for closed-form updates.

### Connection 2: Variational Inference as Approximate EM

In classical EM, the E-step is tractable when the model has conjugate structure or factored
posteriors. In general, computing $p(Z|x;\theta)$ exactly requires a sum or integral over all
possible latent configurations, which is exponentially expensive for many graphical models.

**Variational inference** replaces the exact E-step with an optimisation: choose $q$ from a
restricted family $\mathcal{Q}$ (e.g., fully factored distributions — the **mean field** approximation),
and find the member of $\mathcal{Q}$ that minimises $\text{KL}(q \| p(Z|x;\theta))$.

The mean field assumption says $q(Z) = \prod_j q_j(Z_j)$ — the approximate posterior fully
factorises. This breaks all dependencies between latent variables, which is wrong, but
yields a tractable optimisation. The resulting coordinate ascent updates are called the
**variational EM** or **CAVI** (Coordinate Ascent Variational Inference) algorithm.

The key insight: classical EM is the special case of variational EM where the exact posterior
is in the family $\mathcal{Q}$ (i.e., the model is simple enough that no approximation is needed).
VAEs are the special case where $\mathcal{Q}$ is parameterised by a neural network and optimised
with gradient descent rather than coordinate ascent.

### Connection 3: The ELBO, KL Divergence, and Contrastive Objectives

The ELBO decomposition $\log p(x) = \mathcal{L}(\theta, q) + \text{KL}(q \| p)$ has a direct descendant
in contrastive learning. The **InfoNCE loss** used in SimCLR, CLIP, and related methods
is:

$$\mathcal{L}_\text{InfoNCE} = -\mathbb{E}\!\left[\log \frac{e^{f(x)^T f(x^+)}}{\sum_{x'} e^{f(x)^T f(x')}}\right]$$

The numerator scores the correct (positive) pair; the denominator normalises over all
candidates including negatives. This is structurally analogous to the posterior computation
in the EM E-step: score how well each component explains the data, then normalise across
all components via Bayes' rule. The softmax over similarities plays the same role as the
Bayes denominator — though the objects being scored (learned representations vs. Gaussian
likelihoods) and the purpose (discriminative similarity vs. generative posterior) are different.

The unifying principle: **all of these objectives compare a model's score for the correct
answer against a normalising sum over all alternatives**. EM, VAEs, and contrastive
learning are different instances of the same score-and-normalise pattern, operating at
different levels of the modelling hierarchy.

## Key Takeaways

1. **The core problem** is the log of a sum: $\log \sum_Z p(x, Z; \theta)$ has no closed-form maximiser for most models. EM replaces this intractable objective with a surrogate (the ELBO) that is a tractable lower bound.

2. **The ELBO** $\mathcal{L}(\theta, q) = \mathbb{E}_q[\log p(x,Z;\theta)] + H(q)$ is derived by introducing an auxiliary distribution $q(Z)$ and applying Jensen's inequality. It equals the true log-likelihood minus the KL divergence from $q$ to the true posterior.

3. **The E-step** sets $q = p(Z|x;\theta_t)$ — the true posterior under current parameters. This is derived by maximising the ELBO over $q$ (via Lagrange multipliers) or equivalently by zeroing the KL gap. The ELBO then equals the true log-likelihood exactly at $\theta_t$, and lies strictly below it at any other $\theta$ (because the frozen $q$ no longer matches the posterior there).

4. **The M-step** maximises $Q(\theta|\theta_t) = \mathbb{E}_{Z|x,\theta_t}[\log p(x,Z;\theta)]$ over $\theta$. The entropy term $H(q)$ is constant in $\theta$ and dropped. For exponential family models, this has a closed-form solution.

5. **Monotone likelihood increase** is guaranteed by two independent proofs: (a) the KL proof shows the increase decomposes into the M-step gain plus a non-negative KL term; (b) the ELBO proof shows the E-step tightens the bound, then the M-step climbs it.

6. **GMM example**: the E-step computes responsibilities via Bayes' theorem; the M-step gives responsibility-weighted means, covariances, and mixing weights — all derived by calculus on the $Q$ function.

7. **EM is not globally convergent**: the guarantee is local. Multiple restarts or good initialisation (e.g., $k$-means) are essential in practice. K-means is an effective EM initialiser because it provides hard initial cluster assignments from which $r_{ik}^{(0)}$ (set to 1 for the nearest centroid, 0 otherwise), $\mu_k^{(0)}$, $\Sigma_k^{(0)}$, and $\pi_k^{(0)}$ can all be computed in one pass via the M-step formulas. K-means tends to find well-separated centroids (especially with k-means++ seeding), so the EM run inherits sensible starting assignments rather than the near-random initialisation that random restarts provide. The main risk is that k-means assumes isotropic clusters, so for GMMs with highly anisotropic components, k-means initialisation can still be poor — in that case, random restarts remain important.

8. **The VAE ELBO** is the same object as the EM ELBO. Expanding the VAE KL term shows $\mathcal{L}_\text{VAE} = \mathbb{E}_{q_\phi}[\log p_\theta(x,z)] + H(q_\phi)$ — exactly the EM form. The reconstruction/KL split is just one way of grouping terms; the reconstruction term alone does not equal the M-step objective (it omits $\mathbb{E}[\log p(z)]$), but the full ELBO structure is identical.

9. **Variational inference** generalises the E-step to approximate posteriors from a restricted family $\mathcal{Q}$, optimised by coordinate ascent (mean field / CAVI). Classical EM and VAEs are both special cases.

10. **The unifying principle** connecting EM to modern ML: score the correct answer, normalise over all alternatives. This pattern underlies EM, VAEs, MMI training, InfoNCE, and cross-entropy — a single idea instantiated at progressively larger scale.
