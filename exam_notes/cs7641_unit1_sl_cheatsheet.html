1 В· Decision Trees & ID3

Entropy & Information Gain

H(S) = вҲ’ОЈ pбөў logвӮӮ pбөў (convention: 0В·log0 := 0) IG(S, A) = H(S) вҲ’ ОЈ_{v вҲҲ vals(A)} (|SбөҘ|/|S|) В· H(SбөҘ) ID3 root = argmax_A IG(S, A)
  • ID3 picks highest IG вҖ” NOT the attribute with lowest entropy
  • Conditional entropy = weighted average of per-value subset entropies
  • Pure subsets (all one class) contribute 0 to the weighted sum
  • Ties in IG: random selection, fixed order, or first-in-dataset are all valid
  • Numeric attributes must be discretized or threshold-split before use in ID3
  • Creates one branch per unique value of a categorical attribute
  • Multi-valued attrs can artificially inflate IG вҖ” C4.5 gain ratio corrects this

Leaf Node Properties

  • All examples same label вҶ’ entropy = 0 вҶ’ no further split
  • Path to a leaf may involve many splits along the way
  • ID3 assigns majority class at leaf (does not require вүҘ2 examples)
  • Leaf does NOT mean all features contributed вҖ” many may be unused

Generalization / Avoiding Overfitting

  • Prune branches that overfit training data
  • Limit tree depth or minimum examples per leaf
  • Stop splitting when IG is very low
  • Cross-validate stopping criteria
  • Adding more attributes does NOT guarantee better generalization
  • Never use the label as an input feature

2 В· Boosting & AdaBoost

Core Idea

  • Train weak learners sequentially; each focuses on previously misclassified examples
  • Misclassified examples get higher weight; correctly classified get lower weight
  • Correctly classified examples still included вҖ” just downweighted (not ignored)
  • Weak learner requirement: error Оө < 0.5 (better than random); if Оө вүҘ 0.5, learner is discarded
  • Aggregates with weighted votes вҖ” not equal votes
  • Designed for weak (underfit) base learners вҖ” stumps are typical
  • Can overfit on noisy datasets with too many learners
  • Order per round: train hвӮң вҶ’ compute ОөвӮң вҶ’ compute ОұвӮң вҶ’ update weights. ОұвӮң must be known before weights can move.
  • Why ОұвӮң in the weight update (not ОөвӮң directly): better learner вҶ’ larger ОұвӮң вҶ’ e^{+ОұвӮң} is large вҶ’ misclassified examples boosted aggressively. Near-random learner вҶ’ small ОұвӮң вҶ’ gentle shift. Redistribution is automatically calibrated to how informative the learner was.
  • ОөвӮң=0.5 вҶ’ ОұвӮң=0 вҶ’ learner contributes nothing. ОөвӮң>0.5 вҶ’ ОұвӮң<0 вҶ’ vote flipped (discard in practice).

AdaBoost Formulas

Learner weight: ОұвӮң = ВҪ В· ln((1 вҲ’ ОөвӮң) / ОөвӮң) where ОөвӮң = weighted error of learner t Weight update (unnormalized): misclassified: wбөў' = wбөў В· e^{+ОұвӮң} вҶҗ weight goes up correct: wбөў' = wбөў В· e^{-ОұвӮң} вҶҗ weight goes down Normalize: wбөў вҶҗ wбөў' / Z where Z = ОЈ wбөў'

3 В· Regression & Classification

Polynomial Regression вҖ” Bias / Variance

  • Degree-d polynomial has d+1 parameters (including constant term)
  • Higher degree вҶ’ lower bias, higher variance вҶ’ risk of overfitting
  • n data points: degree nвҲ’1 вҶ’ zero training error (exact interpolation)
  • Higher degree вү  better generalization; degree-3 may not beat degree-2
  • Select model with lowest cross-validation error (not lowest training error)

Error Metrics

MSE = (1/n) ОЈ (y вҲ’ Е·)ВІ вҶҗ penalizes large errors heavily MAE = (1/n) ОЈ |y вҲ’ Е·| L1 norm of errors = ОЈ |eбөў| = n В· MAE L2 norm of errors = вҲҡ(ОЈ eбөўВІ) = вҲҡ(n В· MSE)

L2 Regularization (Ridge)

  • Adds О»В·ОЈwбөўВІ penalty вҶ’ shrinks weights; вҶ‘ bias, вҶ“ variance
  • Helps with multicollinearity
  • Does NOT zero out weights (that's L1 / Lasso)
  • Higher О» вҶ’ higher bias; does NOT lower training error
  • Ridge does NOT increase model complexity

k-NN вҖ” Bias/Variance & Practical Notes

  • Small k вҶ’ low bias, high variance (jagged boundary)
  • Large k вҶ’ high bias, low variance (smooth) вҶ’ can underfit
  • k=1: memorizes training data, poor generalization on noisy data
  • Odd k avoids ties in binary classification
  • Tune k with cross-validation
  • Feature scaling changes distances вҶ’ can change neighbor identities and predictions
  • z-score scaling: equalizes feature contributions (zero mean, unit variance)
  • Distance-weighted voting (wбөў = 1/d) downweights far neighbors вҶ’ less sensitive to outliers
  • L1 (Manhattan) and L2 (Euclidean) can produce different neighbor orderings
  • LOO-CV valid for k-NN вҖ” held-out point excluded from neighbor search, no leakage
  • Increasing k increases bias AND decreases variance вҖ” not both decrease simultaneously

4 В· Neural Networks & Gradient Descent

Gradient Descent Behavior

  • Negative gradient = direction of steepest descent
  • Zero gradient = critical point: could be min, max, or saddle point
  • If gradient=0 but loss not minimal вҶ’ saddle point
  • Momentum: incorporates past gradients вҶ’ helps escape saddle points / shallow minima
  • Large LR вҶ’ overshoot вҶ’ oscillation or divergence, even for convex functions
  • Constant LR does NOT guarantee convergence if too large
  • GD does NOT guarantee global minimum for non-convex (neural net) losses

ReLU Properties

  • ReLU(x) = max(0, x)
  • Mitigates vanishing gradients compared to sigmoid/tanh
  • Sparse activations: outputs 0 for all negative inputs
  • Dead neurons: if input always negative вҶ’ always outputs 0 вҶ’ stops learning
  • ReLU is NOT differentiable at x=0 (discontinuous derivative) вҖ” common trap
  • Deep ReLU nets still require gradient descent вҖ” no closed-form training

Universal Approximation Theorem

A single hidden layer with sufficient width can approximate any continuous function вҖ” doesn't specify how many neurons or how to train efficiently.

XOR with MLP

  • Single-layer perceptron cannot solve XOR (not linearly separable)
  • Need вүҘ1 hidden layer with nonlinear activation
  • 2 hidden neurons sufficient; ReLU or sigmoid both work
  • Linear hidden activation вҶ’ still linear overall вҶ’ cannot learn XOR regardless of depth
  • Sigmoid: vanishing gradients slow training; random init can вҶ’ local minima

Learning Rate Strategies

  • Warm-up: stabilizes training in early epochs
  • LR decay: standard in modern deep learning
  • Cyclical LR: periodically increase and decrease
  • Gradient clipping: pair with high LR to prevent instability
  • Exponentially increasing LR вҶ’ divergence (not faster convergence)
  • Adaptive optimizers (Adam) reduce but don't eliminate the need for schedules

5 В· Ensemble Methods вҖ” Bagging & Boosting

Bagging

  • Train each model on a bootstrap sample (draw n with replacement from n points)
  • Each bootstrap sample contains ~63.2% unique examples on average
  • ~36.8% left out per tree вҶ’ out-of-bag (OOB) examples
  • OOB error: predict each example using only trees for which it was OOB вҶ’ free internal CV estimate
  • OOB error computed on left-out examples вҖ” not all examples; no data leakage
  • Random forests = bagging + feature subsampling per split вҖ” NOT boosting
  • Gradient boosting: each new learner fits the negative gradient of the loss w.r.t. current predictions (= residuals for squared loss)

6 В· Kernel Methods & SVM

Kernel Trick

  • Any PSD kernel K(x,z) = вҹЁПҶ(x), ПҶ(z)вҹ© in some (possibly infinite-dim) feature space (RKHS)
  • Kernel trick: evaluate K(x,z) directly вҖ” never compute ПҶ(x) explicitly
  • SVM decision: f(x) = sign(ОЈбөў Оұбөўyбөў K(xбөў,x) + b)

7 В· PAC Learning

Core Guarantee & Formula

Goal: Pr[ error_D(h) > Оө ] вүӨ Оҙ i.e., with prob вүҘ 1вҲ’Оҙ, error of learned h вүӨ Оө m вүҘ (1/Оө) В· [ ln|H| + ln(1/Оҙ) ]
  • Guarantee is for the learned h, not any arbitrary hypothesis
  • Assumes realizability: true concept вҲҲ H
  • Distribution-free: worst-case guarantee over all distributions
  • PAC does NOT guarantee smallest training error
  • Consistent learner (zero train error) needs sufficient m to ensure low test error
  • Infinite |H| does NOT mean not PAC learnable вҖ” infinite classes with finite VC-dim are learnable (e.g. linear classifiers in в„қбөҲ)
  • Sample complexity grows: linearly in 1/Оө (dominant), logarithmically in 1/Оҙ and |H|

8 В· VC Dimension

Definition

VC-dim(H) = size of the largest set H can shatter (realize all 2вҒҝ labelings). Property of the class only вҖ” independent of data distribution.

Shattering вү  every set of that size. VC-dim=k means вҲғ some set of k points that can be shattered вҖ” NOT that every set of k points can be shattered.

Key Facts & Common Traps

  • Finite VC-dim вҶ” PAC learnable (necessary and sufficient under realizability)
  • Higher VC-dim вү  better generalization вҖ” requires more data to avoid overfitting
  • Same VC-dim вү  same sample complexity (constants can differ)
  • VC-dim depends on hypothesis class expressiveness, NOT just number of input features
  • Axis-aligned rectangle VC-dim=4, but the unit-square corners {(0,0),(0,1),(1,0),(1,1)} CANNOT be shattered вҖ” XOR labeling is unrealizable by any rectangle
  • A shatterable 4-point set requires a "staircase" arrangement, e.g. {(0,0),(1,3),(2,1),(3,4)}
  • VC-dim is independent of whether the learning algorithm is deterministic or randomized
