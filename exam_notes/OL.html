1 · Randomized Optimization

Setup & When to Use

  • Goal: argmax_x f(x) ≡ argmin_x loss(x); all algos here are derivative-free
  • Use when: X is discrete · f is a black box · f is non-differentiable — i.e. gradient methods are unavailable
  • Neighbor N(x) = states "one small change" from x (definition is part of problem setup)
  • Apps: Neural arch. search (AmoebaNet = GA, DARTS = differentiable relaxation), hyperparam tuning, TSP, scheduling, RL with sparse reward
  • If f is continuous & differentiable, use gradients — far more sample-efficient (gradient gives d directional pieces of info per eval; neighbor sampling gives 1)

What randomized opt IS / IS NOT (one-glance summary):

  • IS: uses probabilistic decisions to search — random neighbors (RHC), random uphill acceptance (SA), random selection / mutation / crossover (GA). Randomness is the mechanism for escaping local optima.
  • IS NOT: deterministic, predictable, or "always faster than gradient methods." Different runs ⇒ different paths and possibly different solutions.
  • NO finite-step global-optimum guarantee in practice. Geman–Geman gives SA asymptotic global optimality under logarithmic cooling — but it's impractically slow; geometric SA, RHC, and GA give no finite-time guarantee at all.

Randomized Hill Climbing (RHC)

  • Sample random neighbor → accept only if it strictly improves objective ("improving-only")
  • Derivative-free; uses objective evaluations only
  • Sampling many neighbors per step ⇒ approaches greedy hill climbing
  • Stuck at local optima / plateaus (no admissible move ⇒ stalls)
  • Finite state space does NOT guarantee reaching global optimum
  • Near a local optimum: improving neighbors get rarer ⇒ expected improvement → 0
  • Fix: random restarts, sideways moves

Simulated Annealing (SA)

Idea: RHC stalls at local optima; SA escapes by occasionally accepting worse moves, with probability decaying over time — explore early, exploit late. Metaphor: heat metal so atoms move freely, then cool slowly to settle into the lowest-energy crystal; cool too fast (quench) ⇒ frozen in a defective local minimum.

Metropolis acceptance — "uphill" always means candidate is WORSE: ΔE > 0 (uphill, worse): P(accept) = exp(−ΔE / T) ΔE ≤ 0 (downhill, better): accept always (probability 1) For minimization (cost/loss C): ΔE = C(candidate) − C(current) For maximization (fitness f): ΔE = f(current) − f(candidate) (equivalently: treat −f as a cost)
  • High T → uphill moves likely (exploration); T → ∞ ≈ random walk
  • Low T → behaves greedily (exploitation); T → 0 ≈ RHC
  • Acceptance depends on ΔE (difference), not absolute energies
  • Bigger downhill = exponentially less likely accepted — algo prefers small detours over big plunges
  • Derivative-free; no gradient/Hessian needed
  • Beneficial moves are NOT accepted with exp(+ΔE/T) — they're accepted w.p. 1
  • Track the best ever seen, not just current state — SA can wander away from a great solution

Genetic Algorithms (GA)

Idea: RHC/SA explore one state; GA maintains a population of N candidates and recombines them via crossover. Two parents in different basins can produce a child inheriting good features from each — reaching regions neither parent could find alone.

One generation:

  1. Selection: sample N parents w/ probability ∝ fitness (roulette wheel)
  2. Crossover (prob pc): pair parents, swap substrings → children
  3. Mutation (prob pm per bit): flip random bits for novelty
  4. Replacement: offspring become next generation (+ optional elitism: force-keep top parents)
Single-point crossover at cut k=3: Parent 1: 1 0 1 | 1 0 Parent 2: 0 1 0 | 1 1 Child 1: 1 0 1 1 1 Child 2: 0 1 0 1 0

Variants: tournament selection (pick k random, keep best), two-point / uniform crossover, rank selection, elitism. Tournament is most common in practice — less sensitive to fitness scale than roulette.

Schema Theorem (Holland)

Schema H = pattern w/ fixed bits + wildcards (e.g. 1*0*). A length-l bit string is an instance of 2ˡ schemas — search is implicitly parallel over schemas, not just strings.

o(H) = order = # fixed bits (low o → robust to mutation) δ(H) = defining len = leftmost-rightmost fixed-bit gap (small δ → robust to crossover) Selection (roulette, fitness-proportionate): E[#copies of x] = N · f(x) / Σf Crossover survival (single-point, prob pc): P(survive) = 1 − pc · δ(H) / (l − 1) Mutation survival (bitwise, prob pm): P(survive) = (1 − pm)^o(H) Schema theorem (combined, lower bound): E[m(H,t+1)] ≥ m(H,t) · f̄(H)/f̄ · [1 − pc·δ(H)/(l−1)] · (1−pm)^o(H) ──────── ────────────────── ───────────── selection crossover survive mutation survive
  • Selection (helpful): high-fitness schemas multiply by f̄(H)/f̄ each generation
  • Crossover destruction (hurts): cut in defining region breaks H ⇒ short schemas (small δ) survive better
  • Mutation destruction (hurts): every fixed bit must avoid flipping ⇒ low-order schemas (small o) survive better
  • Building blocks: short, low-order, high-fitness schemas grow exponentially; GA assembles them via crossover into longer high-fitness schemas
  • Building-blocks hypothesis is empirically contested (deceptive functions exist where good substructures don't combine well); the theorem inequality itself holds

2 · Information Theory

Self-Information (the bedrock)

How surprising is event x? The unique function that is monotone-decreasing in P(x), zero when P(x)=1, and additive for independent events is the log:

I(x) = −log P(x) log₂ → bits ln → nats log₁₀ → dits
  • Rare events ⇒ high surprise; certain events ⇒ 0 surprise
  • Independence ⇒ additive: I(x,y) = I(x) + I(y) (this forces the log)
  • Examples (bits): fair coin → 1; loaded die w/ P(6)=0.5 → outcome "6" carries 1 bit (lowest info on that die)

Entropy — average surprise

H(X) = E[I(X)] = −Σₓ P(x) log P(x)

Three equivalent lenses on the same number:

  • Uncertainty — how unpredictable X is
  • Avg info per sample — how much you "learn" each observation
  • Compression bound (Shannon) — min avg bits/symbol; lower H ⇒ better compression
  • Range: 0 (deterministic) ≤ H(X) ≤ log n (uniform = max)
  • Fair coin: H = 1. Fair 6-die: H = log₂6 ≈ 2.585. Loaded die [0.5, 0.1×5]: H ≈ 2.16

Source Coding & Huffman

Designing an encoding = picking a map symbol → bit-string. To transmit "ABC," concatenate code(A)·code(B)·code(C); receiver decodes the bit stream back. Goal: minimize avg bits per symbol L = Σ pᵢ·|code(i)|.

Why variable-length wins: fixed-length (A=00, B=01, C=10) costs 2 bits each. But if P(A)=0.5, give A a 1-bit code and let rarer symbols pay more → lower average. Frequent → short, rare → long. The "ideal" length for symbol i is −log₂ pᵢ (its self-information).

Prefix code constraint: no codeword is a prefix of another (e.g. can't have A=0 AND B=01 — receiving 01 would be ambiguous). Prefix codes are instantaneously decodable: parse a bit-stream symbol-by-symbol with no delimiters. By Kraft's inequality, restricting to prefix codes loses no optimality.

Shannon's source coding theorem — for i.i.d. symbols drawn from distribution P, any uniquely decodable code obeys:

H(X) ≤ L < H(X) + 1 X = one random symbol from the alphabet (a single "A", "B", or "C" — NOT a message like "ABC") L = expected bits per individual symbol

Entropy is the information lower bound: no code averages fewer than H bits to encode each symbol on its own. (For an n-symbol message, total expected bits ≈ n·L ≥ n·H.)

Huffman coding — optimal prefix code, built by repeated merging:

  1. Sort symbols by probability
  2. Merge the two smallest into a super-symbol with summed probability
  3. Repeat until one node (root) remains
  4. Assign 0/1 to branches → codeword = path from root to leaf

Worked example: P(A)=0.5, P(B)=0.2, P(C)=0.2, P(D)=0.1.

Sort: D(0.1) B(0.2) C(0.2) A(0.5) Merge D+B → DB(0.3); Merge C+DB → CDB(0.5); Merge A+CDB → root(1.0) ROOT 0/ \1 A CDB 0/ \1 C DB 0/ \1 D B Codes: A→0, C→10, D→110, B→111 H(X) = −Σ pᵢ log₂ pᵢ ≈ 1.761 bits/symbol ← Shannon lower bound L = 0.5·1 + 0.2·2 + 0.1·3 + 0.2·3 = 1.80 bits/symbol ← Huffman Overhead = L − H ≈ 0.04 bits/symbol (well under 1, per Shannon)

Why L > H here: the distribution is non-dyadic (e.g. −log₂(0.2) ≈ 2.32 isn't an integer), so Huffman has to round lengths up. Dyadic distributions — every pᵢ a negative power of 2 (½, ¼, ⅛, …) — have integer ideal lengths and Huffman hits L = H exactly. (E.g. P = {0.5, 0.25, 0.25}: H = L = 1.5 bits/symbol.)

Joint & Conditional Entropy

Joint: H(X,Y) = −ΣΣ P(x,y) log P(x,y) Conditional: H(Y|X) = Σₓ P(x) · H(Y | X=x) Chain rule: H(X,Y) = H(X) + H(Y|X)

Read chain rule: total uncertainty about (X,Y) = uncertainty about X + remaining uncertainty about Y after seeing X. Pure accounting identity.

  • Conditioning never hurts (on average): H(Y|X) ≤ H(Y), with equality iff X ⊥ Y
  • Beware: for specific x, H(Y|X=x) can exceed H(Y); only the average is bounded
  • Independence ⇒ additivity: X ⊥ Y ⇒ H(X,Y) = H(X) + H(Y); n iid samples carry n·H(X) bits total (linear in n, not sub-)

Mutual Information

I(X;Y) = H(Y) − H(Y|X) (uncertainty reduction) = H(X) − H(X|Y) (symmetric flip) = H(X) + H(Y) − H(X,Y) (inclusion-exclusion) = D_KL(P(X,Y) ‖ P(X)P(Y)) ← key identity

Read: "how much does knowing X tell you about Y?" Equivalently, "how far is the joint from independence?" (via the KL identity).

Why all four forms are equal (chain rule + marginalization):

Chain rule: H(X,Y) = H(X) + H(Y|X) ⇒ H(Y|X) = H(X,Y) − H(X) H(Y) − H(Y|X) = H(Y) − [H(X,Y) − H(X)] = H(X) + H(Y) − H(X,Y) ← inclusion-exclusion By symmetry (swap X↔Y in chain rule), this also equals H(X) − H(X|Y). KL form: expand H(X) + H(Y) − H(X,Y) into log-sums, then use the marginalization trick Σᵧ P(x,y) = P(x) (and likewise for y) to fold single sums into the double sum: = ΣΣ P(x,y)·log P(x,y) − ΣΣ P(x,y)·log P(x) − ΣΣ P(x,y)·log P(y) = ΣΣ P(x,y) · log [ P(x,y) / (P(x)·P(y)) ] = D_KL( P(X,Y) ‖ P(X)·P(Y) )
  • I(X;Y) ≥ 0; = 0 iff X ⊥ Y
  • Symmetric: I(X;Y) = I(Y;X) — a relationship, not a direction
  • Captures any dependence (linear, nonlinear, deterministic) — unlike correlation
  • = Information Gain: ID3's IG(S,A) is exactly I(A;Y) on the node's data
  • MI invariant to sample size — depends only on probabilities, not counts
  • Direction trap (MI ↔ KL): MI is the KL between joint and product-of-marginals. To maximize MI you want this KL large (joint far from independence). Minimizing this KL ⇒ joint → marginals' product ⇒ MI → 0 ⇒ zero shared info. So "minimize KL" and "maximize MI" pull in opposite directions for this specific KL.

KL Divergence

D_KL(P‖Q) = Σₓ P(x) log [P(x)/Q(x)] = E_P[ log P(X)/Q(X) ] = H(P,Q) − H(P) (cross-entropy − entropy) Cross-entropy: H(P,Q) = −Σₓ P(x) log Q(x)

Operational meaning: extra bits to encode P-samples using Q's codebook (vs. P's optimal codebook) — the "price of model misspecification."

  • D_KL ≥ 0; = 0 iff P = Q (Gibbs' inequality)
  • NOT symmetric: D_KL(P‖Q) ≠ D_KL(Q‖P) — direction matters
  • NOT a metric — no triangle inequality. It's a divergence, not a distance
  • D_KL(P‖Q) penalizes Q missing P's mass (mass-covering); D_KL(Q‖P) penalizes Q putting mass where P=0 (mode-seeking) — VI uses the latter
  • Cross-entropy ≡ classification loss: minimizing H(true, predicted) ≡ minimizing D_KL(true ‖ predicted), since H(true) is fixed

Reading MI as a fraction

I(X;Y) / H(Y) = share of label uncertainty resolved by observing X. If X⊥Y this is 0; if X perfectly determines Y this is 1.

Trick when computing H(Y|X): for each x, condition on it (P(y|x) = P(y,x)/P(x)) — H(Y|X=x) is the entropy of that conditional distribution, not the joint. Then take the weighted average Σₓ P(x)·H(Y|X=x).

3 · Adam Family Optimizers

SGD's Two Problems (the motivation)

Plain SGD: θ ← θ − α · g. Two fundamental weaknesses:

  • Ill-conditioning: for L = θ₁² + 100·θ₂² (Hessian eigenvalues 2 & 200 ⇒ condition number 100), SGD zigzags down θ₂'s steep walls while crawling along θ₁'s gentle floor. Same LR for every coordinate — dominated by the worst direction.
  • Noisy gradients: stochastic minibatches make g_t bounce around its true mean ⇒ wasted oscillation in any direction.

Two independent fixes, each its own thread of innovation:

  • Thread 1 — Adaptive LR (AdaGrad → RMSProp): per-coordinate step sizes that adapt to local curvature. Fixes ill-conditioning.
  • Thread 2 — Momentum (Heavy-ball → NAG): smooth gradients across steps. Fixes oscillation.

Adam = Thread 1 + Thread 2, with bias correction to handle cold start. AdamW fixes a regularization wart in Adam.

Thread 1 — AdaGrad & RMSProp (curvature)

Big idea: coordinates with consistently large |g| are high-curvature directions; shrink their step. Coordinates with small |g| can take bigger steps. Squared gradients over time proxy diagonal curvature.

AdaGrad: G_t = Σᵢ≤ₜ gᵢ² (cumulative — never forgets) θ ← θ − α · g_t / (√G_t + ε) RMSProp: v_t = ρ·v_{t−1} + (1−ρ)·g_t² (EMA, ρ≈0.9–0.99) θ ← θ − α · g_t / (√v_t + ε)
  • AdaGrad flaw: G_t monotonically grows ⇒ effective LR vanishes late in training
  • RMSProp fix: EMA forgets old gradients ⇒ tracks nonstationary curvature, no vanishing
  • Why √v_t? v ≈ E[g²] (variance); √ converts back to gradient-magnitude scale so 1/√v has units of 1/|g|

The Preconditioning Lens

All of these can be written as θ ← θ − α · P_t · g, where P_t scales each coordinate. The choice of P_t is the algorithm:

Thread 2 — Momentum & NAG (smoothing)

Big idea: instead of stepping by the current noisy gradient, step by a smoothed version (EMA of recent gradients). Consistent directions accumulate; alternating ones cancel. Rolling ball with mass — past direction lingers; only sustained forces redirect it.

Heavy-ball (EMA-style momentum): v_t = β·v_{t−1} + (1−β)·g_t (EMA of gradients, β ≈ 0.9) θ ← θ − α · v_t NAG (Nesterov): v_t = β·v_{t−1} + (1−β)·∇L(θ_t + β·v_{t−1}) ← "look-ahead" gradient θ ← θ − α · v_t
  • Heavy-ball: amplifies consistent directions, damps zigzag in narrow valleys
  • NAG: peeks ahead to where momentum will take you, then computes the gradient there — gets to course-correct before committing
  • NAG: O(1/t²) convergence on smooth convex functions (vs GD's 1/t)
  • Trap: O(1/t²) does NOT extend to nonconvex deep networks. NAG still helps empirically (saddle-point escape, local geometry), but not for that theoretical reason.
  • Both are first-order: just EMAs of gradients. NO Hessian or second-derivative info required.

Adam = Thread 1 + Thread 2

Adam: m_t = β₁·m_{t−1} + (1−β₁)·g_t ← momentum (Thread 2) v_t = β₂·v_{t−1} + (1−β₂)·g_t² ← RMSProp (Thread 1) m̂_t = m_t / (1 − β₁ᵗ) ← bias correction v̂_t = v_t / (1 − β₂ᵗ) θ ← θ − α · m̂_t / (√v̂_t + ε) ───── ───── smoothed g diagonal P_t Defaults: β₁ = 0.9, β₂ = 0.999, ε = 10⁻⁸

So Adam is literally "momentum on the numerator, RMSProp on the denominator, bias correction on both." Each piece earns its place by solving one of SGD's two problems.

What Bias Correction Actually Does

At t=1 with m₀=v₀=0: m₁ = (1−β₁)·g ≈ 0.1·g, v₁ = (1−β₂)·g² ≈ 0.001·g² — both shrunk because EMAs start cold. Dividing by (1−βᵗ) un-biases them: m̂₁ = g, v̂₁ = g², so step ≈ α·sign(g).

Net effect: the very first step has full magnitude ≈ α (raw m₁ alone would give ≈ α/30, too small). Later steps shrink as √v̂ catches up to gradient scale.

L2 Penalty vs Weight Decay — preconditioner-eye view

L2 reg adds ½λ‖θ‖² to the loss ⇒ modified gradient = ∇L + λθ. Plug into the preconditioned step:

θ ← θ − α · P_t · (∇L + λθ) = θ − α·P_t·∇L − α·λ·P_t·θ ───────── decay is preconditioned too!
  • SGD (P = c·I): decay = α·λ·c_t·θ ⇒ uniform shrinkage ⇒ L2 ≡ weight decay ✓
  • Adam (P diagonal, non-scalar): decay = α·λ·P_t·θ ⇒ coordinate-wise. Coords w/ small g get aggressive decay; coords w/ large g barely decay ⇒ L2 ≠ weight decay
  • Equivalence theorem: L2 ≡ uniform decay iff P_t = c_t · I (scalar × identity)

AdamW: Decouple the Decay

Vanilla Adam + L2 (broken): θ ← θ − α · P_t · (∇L + λθ) ← decay sandwiched in P_t AdamW (fixed): θ ← θ − α · P_t · ∇L − α · λ · θ ← decay applied OUTSIDE P_t
  • Decay yanked out of the preconditioner sandwich ⇒ uniform shrinkage restored
  • α and λ become independent knobs (with vanilla Adam they coupled through P_t)
  • Rescale λ when steps-per-epoch changes (more updates/epoch = more total decay)
  • Exclude biases & LayerNorm/BatchNorm scale-shift from decay (best practice)

What AdamW changes vs keeps (one-glance):

  • Only one thing changes: where weight decay is applied (decoupled −α·λ·θ instead of L2-in-the-gradient).
  • Everything else is identical to Adam: momentum m_t (β₁), variance v_t (β₂), bias correction m̂/v̂, ε. Default β₁=0.9, β₂=0.999, ε≈10⁻⁸ all carry over unchanged.
  • λ = 0 ⇒ AdamW ≡ Adam exactly (the −α·λ·θ term vanishes; the rest is Adam).
  • AdamW does NOT remove bias correction (unrelated concern), does NOT add λθ to the gradient (that's the broken behavior it fixes), and does NOT guarantee convergence to a global optimum on nonconvex problems (no first-order optimizer does).

4 · Bayesian Learning

Bayes' Rule — the four ingredients

The principled answer to "I had a belief, then I saw data — how should belief update?" Falls straight out of the chain rule: P(H,D) = P(H|D)·P(D) = P(D|H)·P(H).

Why "principled" (Cox / Dutch book): Bayes' rule is the unique consistent way to update degrees of belief that respects the probability axioms — alternatives lead to incoherent (Dutch-book-able) bets.

What Bayesian inference gives you (vs frequentist):

  • Full posterior distribution over h, not a point estimate — supports credible intervals, posterior predictive, Bayes factors, model averaging
  • Integrates prior knowledge (domain info, regularization) with data via posterior ∝ likelihood × prior
  • NOT always unimodal — multimodal priors / non-identifiable likelihoods / symmetry give multimodal posteriors. Capturing that multimodality is a feature, not a bug.
  • NOT computationally trivial — only conjugate priors give closed-form posteriors. Otherwise: MCMC, variational inference, or Laplace approximation. "Principled" ≠ "easy to compute."
  • NEVER discards the prior — that's MLE, not Bayes. The prior is the Bayesian distinguishing feature.

Mantra: posterior ∝ likelihood × prior

Three Useful Forms

Full: P(h|D) = P(D|h)·P(h) / P(D) Proportional: P(h|D) ∝ P(D|h)·P(h) ← drop P(D) for argmax/comparing Odds: P(h₁|D)/P(h₂|D) = [P(h₁)/P(h₂)] · [P(D|h₁)/P(D|h₂)] posterior odds = prior odds × Bayes factor (LR) ↑ P(D) cancels in numerator/denominator

MLE vs MAP — "regularization is a prior"

  • MLE: argmax_h P(D|h) — best explanation, ignores prior
  • MAP: argmax_h P(D|h) · P(h) — explanation weighted by prior plausibility
  • MAP collapses to MLE under uniform prior (constant P(h) doesn't affect the argmax)
  • Regularization = prior in disguise: L2 ≡ MAP w/ Gaussian prior on weights; L1 ≡ Laplace prior. Reg strength ≡ prior tightness.
  • "More probable hypothesis" = posterior, NOT likelihood. Higher P(D|h) ⇒ higher P(h|D) only under uniform prior. With an informative prior, a high-likelihood overfit-prone h can lose to a medium-likelihood plausible one. Likelihood ≠ probability of h; it's probability of D given h.

Common Bayesian Fallacies

  • Prosecutor's fallacy: P(D|h) ≠ P(h|D) — converting requires Bayes' rule + a prior
  • LR = 1 ≠ favors h₁ — data is uninformative; posterior odds = prior odds (no update)
  • Prior = 0 ⇒ posterior = 0 forever — no data resurrects a zeroed prior; dogmatic priors are unfalsifiable
  • Likelihood = 0 ⇒ posterior = 0 — if your model says D is impossible, the model dies on observing D (motivates Laplace smoothing)

Bayes Decision Rule for Classification

For label y given features x, classify by the posterior:

ŷ(x) = argmax_y P(y | x) = argmax_y P(x | y) · P(y) ───────── ──── likelihood prior P(x) cancels — same denominator for every class, so it drops out of the argmax.
  • Equal priors (binary): predict y=1 iff P(y=1|x) > 0.5 ⇔ P(x|y=1) > P(x|y=0). Classification is then pure likelihood comparison.
  • Prior monotonicity: with likelihoods fixed, raising P(y=1) raises P(y=1|x) monotonically — the prior shifts the decision threshold toward the favored class.
  • Class imbalance ≠ uniform priors. The Bayes-optimal classifier wants the true frequencies. Setting priors uniform throws away base-rate info ⇒ over-predicts the minority class. Fix imbalance via resampling, cost-sensitive loss, or threshold tuning — NOT by lying about priors.

LDA / QDA — Gaussian likelihoods

LDA = Linear Discriminant Analysis  ·  QDA = Quadratic Discriminant Analysis (named for the boundary shape they produce). Both are generative classifiers: model P(x | y) = 𝒩(μ_y, Σ_y), then apply Bayes. The log-posterior-ratio determines the boundary shape:

log [P(y=1|x) / P(y=0|x)] = log[P(x|y=1)/P(x|y=0)] + log[P(y=1)/P(y=0)] = −½(x−μ₁)ᵀ Σ₁⁻¹ (x−μ₁) + ½(x−μ₀)ᵀ Σ₀⁻¹ (x−μ₀) + const Shared covariance (Σ₀ = Σ₁ = Σ): quadratic terms cancel ⇒ LINEAR in x → LDA Different covariances: quadratic terms survive ⇒ QUADRATIC in x → QDA
  • LDA models a full shared Σ across classes; Gaussian NB w/ shared diagonal Σ is a restricted form of LDA. Logistic regression is the discriminative analog of LDA (same hyperplane boundary, different fitting objective).
  • QDA: per-class Σ; boundary is a conic section (parabola/ellipse/hyperbola in 2D)

Bayes-Optimal Classifier — the irreducible floor

The classifier you'd build if you knew the true P(X,Y) — provably the best possible. No model can do better in expectation.

Bayes rule: h*(x) = argmax_y P(Y=y | X=x) Bayes boundary: {x : P(Y=1|X=x) = 0.5} (binary) Bayes error: E_X[ 1 − max_y P(Y|X=x) ] (irreducible)
  • Bayes error = 0 ⇒ classes perfectly separable; > 0 ⇒ inherent overlap (oracle is wrong sometimes)
  • Bias-variance: total error = Bayes error + bias² + variance
  • 1-NN asymptotic error ≤ 2× Bayes error (Cover & Hart)

Noise & the Bayes Boundary

  • Noise always raises Bayes error
  • Symmetric label noise (rate η each direction): preserves ranking of P(y|x) ⇒ boundary location unchanged; only Bayes error rises
  • Asymmetric / feature noise: shifts the boundary AND raises Bayes error

5 · Bayesian Networks (DAGs)

Why DAGs? The parameter explosion

Joint over n binary variables = 2ⁿ − 1 parameters. n=100 ⇒ 10³⁰. Infeasible to store, learn, or even write down.

BN exploits structure: most variables are conditionally independent of most others, given a few key parents. The DAG encodes which variables matter for which.

Factorization

P(X₁, …, Xₙ) = Πᵢ P(Xᵢ | Pa(Xᵢ))

Each variable depends only on its parents. Given parents, X_i is independent of all its non-descendants. Each CPT is a local model; the DAG glues them into a global joint.

Canonical Example: Burglary Network

Burglary Earthquake (independent causes) \ / v v Alarm / \ v v JohnCalls MaryCalls P(B,E,A,J,M) = P(B)·P(E)·P(A|B,E)·P(J|A)·P(M|A) Param count: 1 + 1 + 4 + 2 + 2 = 10 (vs 31 for full joint)

Three Building-Block Patterns

PatternFormDefault (no cond.)Conditioning on B
ChainA → B → CA,C correlatedBLOCKS  (A ⊥ C | B)
ForkA ← B → CA,C correlated (common cause)BLOCKS  (A ⊥ C | B)
ColliderA → B ← CA ⊥ C by defaultOPENS! ("explaining away")
  • Chain: Smoking → Tar → Cancer. Knowing Tar makes Smoking irrelevant for Cancer prediction.
  • Fork: Cold → Cough; Cold → Fever. Cough & Fever correlated through cold; given the cold, independent.
  • Collider: Burglary, Earthquake → Alarm. B,E independent in population. Hear Alarm + learn Earthquake → Burglary becomes less likely (the earthquake "explains" the alarm).

The collider reversal is the most counterintuitive thing in BNs. Conditioning creates dependence between previously-independent parents. Mechanism: a collider's two parents now compete to explain the observed effect.

D-Separation Rule

A path X — … — Y is blocked by conditioning set Z if it contains:

  • A non-collider (chain or fork node) that is in Z, OR
  • A collider such that neither it nor any descendant is in Z

X ⊥ Y | Z iff every path between X and Y is blocked.

Local Markov Property

Each Xᵢ ⊥ non-descendants(Xᵢ) | Pa(Xᵢ) — equivalent to the factorization, easier to apply locally.

Burglary network example: JohnCalls ⊥ Earthquake | Alarm (Earthquake is a non-descendant of JohnCalls; Alarm is JohnCalls's parent).

Ancestral Sampling — drawing from the joint

To draw an i.i.d. sample from P(X₁,…,Xₙ): traverse the DAG in topological order (parents before children); for each node sample Xᵢ ~ P(Xᵢ | Pa(Xᵢ)) using the parent values already drawn. The chain rule executed forward.

  • Exact i.i.d. draws from the joint — no MCMC, no rejection. One pass per sample.
  • Uses only the forward CPTs P(child | parent); never P(parent | child) or any inverse
  • LLN: empirical frequency of any event over n samples → its true probability ⇒ Monte Carlo gives marginals/expectations without analytic integration
  • Ignoring an edge breaks sampling two ways — e.g. for A → B, drawing B from a fair coin instead of P(B|A) gives (i) wrong marginal P(B=1) ≠ Σₐ P(B=1|a)·P(a), AND (ii) wrong dependence structure: generated A,B end up independent, but the true joint has A,B dependent through the edge. "Independent A,B" ≠ "i.i.d. from the joint P(A,B)".

Other Properties

  • DAG required — directed cycles forbidden (would break factorization)
  • CPTs can be any valid conditional (categorical, Gaussian, logistic, NN, …) — Gaussianity NOT required

Naive Bayes — the simplest BN classifier

Structure: class y is the parent of every feature x_j; no edges among features. The DAG is the conditional-independence assumption.

y / | \ v v v x₁ x₂ … x_d P(y, x) = P(y) · Πⱼ P(xⱼ | y) ← "naive" CI assumption Decision rule: ŷ = argmax_y P(y) · Πⱼ P(xⱼ | y) Log-space: ŷ = argmax_y [ log P(y) + Σⱼ log P(xⱼ | y) ] ← avoids underflow
  • Why factor: full P(x | y) needs exponentially many params in d; d univariate P(xⱼ | y) is cheap to estimate from limited data
  • Zero-likelihood trap: if (xⱼ, y) never co-occurred in training, P(xⱼ | y) = 0 zeros the entire product. Laplace smoothing (add 1 to every count; Dirichlet generalizes) prevents this
  • Robustness paradox: even when CI is wildly violated, NB often classifies correctly. Posterior probabilities get miscalibrated (correlated features double-counted), but the argmax ranking of classes typically survives — and classification only needs the argmax
  • Gaussian NB: P(xⱼ | y) = 𝒩(μ_jy, σ²_jy) for continuous features. With shared Σ across classes and uncorrelated features, Gaussian NB ⇒ restricted LDA (cf. §4)
  • NB decision rule uses all features in the product — NOT just the single largest likelihood
