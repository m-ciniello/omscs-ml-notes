1 · Markov Decision Processes

The MDP Five-Tuple

⟨S, A, T, R, γ⟩ S = states A = actions T(s, a, s') = transition probability (sum to 1 over s') R(s, a) = expected reward γ ∈ [0,1] = discount Markov property: P(s_{t+1} = s' | s_t, a_t, s_{t-1}, …) = T(s_t, a_t, s') ⇒ next state depends ONLY on (s, a), not on history Return: G_t = Σ_{k=0}^∞ γᵏ · r_{t+k+1} (discounted sum of future rewards)
  • Deterministic env: for each (s,a), exactly one s' has T=1
  • Stochastic env: multiple s' have non-zero T (noise / slippage)
  • Reward lives on the edge, not the destination: R(s, a) is paid once, at the moment of taking action a in s. V(terminal) = 0 always — terminal states have no future. Common bug: setting V(G) = "the reward of reaching G" ⇒ each backup re-injects γ·V(G), inflating values geometrically forever.
  • γ → 0: myopic; γ → 1: far-sighted. Sparse rewards ⇒ slow learning; shaping helps but bad shaping changes π*.

Policies

  • Deterministic π(s) = a: single action per state with prob 1
  • Stochastic π(a|s): distribution over actions; allows exploration / hedging
  • Optimal policy π* maximizes expected return from every state
  • π* may not be unique — multiple policies can achieve V*(s)
  • A uniform-random policy is generally not optimal — it fails to exploit learned values

Value Functions & Bellman Equations

Core intuition: Q is V with the first action forced. V^π(s) = "follow π forever from s"; Q^π(s, a) = "force action a at s, then revert to π." Optimal versions Q*, V* swap π for π*.

SymbolPlain English
V^π(s)Expected discounted return: start at s, follow π forever.
Q^π(s, a)Same as V^π but force action a at s; then revert to π. (a need not be what π picks.)
V*(s)V^π under the best policy π*.
Q*(s, a)Q^π under π*: force a now, act optimally after.
π*(s)argmax_a Q*(s, a) — the action where V* and Q* agree.

V ↔ Q identities (how PI / VI / Q-learning move between the two views):

V^π(s) = Σ_a π(a|s) · Q^π(s, a) (general — V is Q averaged over π's choice) ↓ deterministic π collapses the sum V^π(s) = Q^π(s, π(s)) (V = Q at π's chosen action) ↓ optimal π picks the max V*(s) = max_a Q*(s, a) (V* = Q at the BEST action) π*(s) = argmax_a Q*(s, a) (π* = which action achieves it)
Bellman EXPECTATION (deterministic π): V^π(s) = R(s, π(s)) + γ · Σ_{s'} T(s, π(s), s') · V^π(s') Bellman EXPECTATION (stochastic π — e.g. uniform random, ε-greedy): V^π(s) = Σ_a π(a|s) · [ R(s, a) + γ · Σ_{s'} T(s, a, s') · V^π(s') ] ───────────── outer Σ_a averages over the policy's own randomness Bellman OPTIMALITY (for π*): V*(s) = max_a [ R(s, a) + γ · Σ_{s'} T(s, a, s') · V*(s') ] Q*(s,a) = R(s, a) + γ · Σ_{s'} T(s, a, s') · max_{a'} Q*(s', a')
  • Expectation form (no max, π fixed) ⇒ linear system in V^π — solvable directly. Used in PI eval step and SARSA target.
  • Optimality form (max over a) ⇒ nonlinear — must iterate. Used in VI and Q-learning target. Any policy greedy w.r.t. Q* is optimal.
  • Why the distinction matters: the form determines the algorithm. PI cleverly uses Expectation (easy, linear) for inner eval and Optimality (max) only for the outer improvement; VI uses Optimality directly. Same Expectation/Optimality split recurs in model-free RL as on-policy (SARSA) vs off-policy (Q-learning).
  • Changing T or R changes V* ⇒ may change π*

What Bellman IS / IS NOT for (one-glance):

  • IS: recursive decomposition of the value function — relates V(s) to V(s') across one transition. Computes expected return under a given π (Expectation form) or under π* (Optimality form). Provides the structural recursion that PI/VI exploit.
  • IS NOT: a way to estimate rewards R or transition probabilities T — those are MDP inputs, given. Does NOT directly produce π* — values come first, then π*(s) = argmax_a Q*(s, a) reads it off. Bellman is the equation, not the algorithm; PI/VI do the iteration.

Self-Loop Trap

If an action keeps the agent in s, the Bellman RHS still has γ·V(s). Don't drop the recursive term — you'd toss the entire infinite discounted future of staying put. Common triggers: stochastic policy (Stay w/ prob p, Go w/ prob 1−p), wall-bump in a grid world, absorbing state reached probabilistically.

Bellman: V(s) = (1−p)·r_g + p·(r_s + γ·V(s)) ← V(s) on BOTH sides (self-loop) Solve for V(s): V(s)·(1 − pγ) = (1−p)·r_g + p·r_s ⇒ V(s) = [(1−p)·r_g + p·r_s] / (1 − p·γ) ← (1 − pγ)⁻¹ = geometric-series sum of all discounted "Stay" steps

Policy Iteration (PI)

Planning (needs ⟨T, R⟩). Central object: π — V^π is recomputed each outer round to evaluate the current policy. Uses Expectation form for eval (linear, π fixed) and Optimality form only for the improve step.

Init π₀ arbitrarily. Repeat: (1) EVALUATION — solve V^π exactly: V^π(s) = R(s, π(s)) + γ · Σ_{s'} T(s, π(s), s') · V^π(s') ∀s (linear system in |S| unknowns — O(|S|³) — or inner Bellman-expectation loop) (2) IMPROVEMENT — greedy w.r.t. V^π: π'(s) = argmax_a [ R(s, a) + γ · Σ_{s'} T(s, a, s') · V^π(s') ] Stop when π' = π ⇒ π* found.
  • Monotone: V^{π_{k+1}} ≥ V^{π_k} pointwise — never gets worse
  • Finite-step termination: finite MDPs converge in ≤ |A|^|S| iters (usually a handful in practice)
  • Expensive per iteration — evaluation solves a full linear system or runs an inner Bellman loop to convergence

Value Iteration (VI)

Planning (needs ⟨T, R⟩). Central object: V — no π is stored during the loop; π* is derived once at the end via argmax. Uses Optimality form (max) directly on V every sweep.

Init V₀(s) = 0 ∀s. Repeat until ‖V_{k+1} − V_k‖_∞ < ε: V_{k+1}(s) = max_a [ R(s, a) + γ · Σ_{s'} T(s, a, s') · V_k(s') ] ∀s Recover policy: π*(s) = argmax_a [ R(s, a) + γ · Σ_{s'} T(s, a, s') · V*(s') ]
  • Cheap iterations — one max-Bellman sweep, no inner eval loop
  • γ-contraction: ‖V_{k+1} − V*‖_∞ ≤ γ · ‖V_k − V*‖_∞ ⇒ exponential at rate γ (slower when γ ≈ 1)
  • Greedy policy stabilizes before V does — argmax often locks in early while V_k is still rising; π* found long before V converges
  • VI as discounted reverse-BFS: R(s, a) is the seed paid once on a transition; every other Bellman backup reduces to V(s) ← max_a γ · V(s'), copying value one γ-discounted hop per iter outward from reward sources. Sparse R ⇒ # iters ≈ MDP diameter; shaping densifies seeds; large γ extends echo range.

Modified Policy Iteration = approximate eval with a few Bellman sweeps (not full convergence). Bridges PI ↔ VI; usually fastest in practice.

Common traps: Neither always converges faster — PI has fewer but more expensive iterations; VI has many cheap ones; total time is problem-dependent. Both PI and VI are planning algorithms requiring ⟨T, R⟩ — neither is model-free. Model-free RL (Q-learning, SARSA, MC) is a different category that learns from samples without a model.

Worked Example — PI vs VI on the Haunted House

Setup: S = {L, Q}, A = {organ, incense}, γ = 0.9. Deterministic transitions:

state \ actionorganincense
L (laughing)→ Q (r = +1)→ L (r = −1)
Q (quiet)→ L (r = −1)→ Q (r = +1)

VI — only V is stored; π derived once at the end:

iter V(L) = max(organ, incense) V(Q) = max(organ, incense) 0 0 0 1 max(1+0, −1+0) = 1 max(−1+0, 1+0) = 1 2 max(1+0.9, −1+0.9) = 1.9 max(−1+0.9, 1+0.9) = 1.9 3 max(1+1.71, −1+1.71) = 2.71 max(−1+1.71, 1+1.71) = 2.71 ∞ → 10 → 10 Final π*: π*(L) = argmax(1+0.9·10, −1+0.9·10) = organ ; π*(Q) = incense

PI — only π is stored; V recomputed each round. Start π_0 = (L: incense, Q: incense).

ITER 1 — Eval (Expectation form: π fixed; both states self-loop) V(L) = −1 + 0.9·V(L) ⇒ V(L) = −10 V(Q) = 1 + 0.9·V(Q) ⇒ V(Q) = +10 ITER 1 — Improve (compute Q^π(s, a) = R(s, a) + γ · V^π(s'), then argmax_a) Q(L, organ) = 1 + 0.9·V(Q) = 1 + 0.9·(10) = 10 ←─┐ Q(L, incense) = −1 + 0.9·V(L) = −1 + 0.9·(−10) = −10 │ argmax ⇒ FLIP π(L) to organ Q(Q, organ) = −1 + 0.9·V(L) = −10 Q(Q, incense) = 1 + 0.9·V(Q) = 10 ←─ argmax ⇒ keep π(Q) = incense ITER 2 — Eval under π_1 = (organ, incense) V(Q) = 1 + 0.9·V(Q) ⇒ V(Q) = 10 V(L) = 1 + 0.9·10 = 10 ITER 2 — Improve (Q^π re-computed under new V): no argmax changes ⇒ CONVERGED in 2 outer iters.
  • VI: V refines slowly (1 → 1.9 → 2.71 → … → 10) but the implicit argmax is already π* from iter 1 — VI just doesn't track it
  • PI: π flips at iter 1, locks in at iter 2 — only 2 rounds, but each needed a linear-system solve
  • "Greedy stabilizes before V does" made concrete: at VI iter 1, V(L) = 1 (way off from true 10) yet the greedy choice is already π*

Worked Example — Q^π Under Two Policies (Exploration vs Exploitation)

Setup (γ = 1, 1-step lookahead): at s, two actions — a1 (r = 1, → s'_1) or a2 (r = 0, → s'_2). Each next state offers two further (stochastic) actions whose leaf rewards are 50/50:

s'_1 : a'_1 → {1, 2} E[r] = 1.5 ; a'_2 → {1, 2} E[r] = 1.5 s'_2 : a'_2 → {1, 2} E[r] = 1.5 ; a'_3 → {0, 6} E[r] = 3.0

How the formula collapses to the calc. The cheat-sheet equation is for V^π (outer Σ_a). Here we want Q^π (first action fixed) — so the policy-averaging Σ_{a'} π(a'|s') resurfaces one level deeper than s, hidden inside V^π(s'). Tracing Q(s, a1):

Q^π(s, a1) = R(s, a1) + γ · Σ_{s'} T(s, a1, s') · V^π(s') [Q form: a1 fixed, no Σ_a] = R(s, a1) + γ · V^π(s'_1) [T deterministic ⇒ s'_1] = 1 + V^π(s'_1) [γ = 1] V^π(s'_1) = Σ_{a'} π(a'|s'_1) · Q^π(s'_1, a') [V ↔ Q recursion] = 0.5 · Q^π(s'_1, a'_1) + 0.5 · Q^π(s'_1, a'_2) [uniform π = 1/2] ⇒ Q^π(s, a1) = 1 + 0.5·Q^π(s'_1, a'_1) + 0.5·Q^π(s'_1, a'_2) └─────────── = V^π(s'_1) ──────────┘ stochastic-π averaging Σ_{a'} π(a'|s') surfaces here, NOT at s itself

Where the sums live in this problem:

LevelSum present?Why
at s (first action)No Σ_aa1 fixed — that's what makes it Q, not V
s → s'No Σ_{s'}Transitions are deterministic in this problem
at s' (second action)Yes — 0.5·Q + 0.5·Q= V^π(s'_1) under uniform π
s' → leafNo Σ_{s''}1-step horizon ⇒ Q^π(s', a') = E[leaf reward]

Total exploration (π(a'|s') = 0.5 uniform — Bellman expectation, stochastic form):

Q(s, a1) = R(s, a1) + 0.5·Q(s'_1, a'_1) + 0.5·Q(s'_1, a'_2) = 1 + 0.5·1.5 + 0.5·1.5 = 2.50 Q(s, a2) = R(s, a2) + 0.5·Q(s'_2, a'_2) + 0.5·Q(s'_2, a'_3) = 0 + 0.5·1.5 + 0.5·3.0 = 2.25 ⇒ Pick a1. Averaging across sub-actions dilutes a'_3's high mean.

Greedy (π(a'|s') = argmax_{a'} Q — at a 1-step horizon this is Bellman optimality):

At s'_1: Q(s'_1, a'_1) = Q(s'_1, a'_2) = 1.5 ⇒ either (tied) At s'_2: Q(s'_2, a'_3) = 3.0 > Q(s'_2, a'_2) = 1.5 ⇒ pick a'_3 Q(s, a1) = 1 + 1.5 = 2.50 Q(s, a2) = 0 + 3.0 = 3.00 ⇒ Pick a2. Committing to a'_3 unlocks its full mean.
  • Same MDP, different π ⇒ different Q^π. Q^π(s, a) = R(s, a) + γ · V^π(s'); the future term is V^π, not V*. "Is action a good?" has no answer without committing to a follow-on policy.
  • Uniform π applies Bellman expectation with π(a|s) = 1/k ⇒ averages over sub-actions ⇒ high-mean leaves get diluted by their middling siblings.
  • Greedy π applies argmax everywhere ⇒ at a 1-step horizon V^π_greedy(s') = max_{a'} Q(s', a') (Bellman optimality). Switching π flipped the answer: a1 wins under exploration, a2 wins under greedy.

Q-Learning — off-policy, optimistic target

Target uses MAX over next actions (assumes optimal future): Q(s,a) ← Q(s,a) + α [ r + γ · max_{a'} Q(s',a') − Q(s,a) ] ───────────── greedy bootstrap Off-policy: learns about π* regardless of behavior policy.
  • Off-policy: behavior policy (often ε-greedy) explores; target evaluates the greedy policy
  • Q-values converge to Q* (optimal), even though behavior is exploratory
  • Convergence requires: all (s,a) visited infinitely often (sufficient exploration) AND decaying α (Robbins–Monro). Pure greedy + fixed α ⇒ no convergence guarantee.
  • Q-values do NOT represent return under behavior policy — they reflect optimal future

SARSA — on-policy, realistic target

Target uses the ACTUAL next action a' sampled from current policy: Q(s,a) ← Q(s,a) + α [ r + γ · Q(s',a') − Q(s,a) ] ────────── uses sampled a' (incl. exploration) SARSA = (s, a, r, s', a') tuple — name from the update.
  • On-policy: learns Q for the policy it's actually following (including exploration)
  • More conservative near risk: if exploring near a cliff sometimes triggers ε-random falls, SARSA's Q reflects that danger ⇒ safer learned policy. Q-Learning, optimistic, walks the cliff edge.
  • Classic intuition: "Q-Learning ignores exploration cost; SARSA pays for it."

ε-Greedy Action Selection

With prob 1 − ε: pick greedy (argmax_a Q(s,a)) With prob ε: pick uniformly at random over all N actions If k actions tie for max: P(each greedy) = (1 − ε) / k + ε / N P(each non-greedy) = ε / N
  • ε too high: too random ⇒ ignores learned Q ⇒ slow convergence, high variance
  • ε too low / 0: insufficient exploration ⇒ may never visit some (s,a) ⇒ stuck at local opt
  • Persistent ε > 0 keeps exploration alive after convergence — useful in non-stationary envs
  • Annealing schedule: high ε early (explore), decay over time (exploit)
  • Softmax alternative: P(a) ∝ exp(Q(s,a)/τ) — graded by Q-values, not fixed ε; τ→0 greedy, τ→∞ uniform

3 · Game Theory — Strategies & Equilibrium

Setup

  • Players simultaneously choose actions; each gets a payoff depending on everyone's choices
  • Pure strategy: a single action with prob 1
  • Mixed strategy: a probability distribution over actions (deliberately randomize)
  • Best response: action(s) maximizing your payoff given the opponent's strategy

Nash Equilibrium

A profile of strategies where no player can unilaterally improve by deviating. "Stable" — given everyone else's choice, you're already best-responding.

  • Existence (Nash 1950): every finite game has at least one mixed-strategy Nash equilibrium
  • Pure-strategy Nash may not exist (e.g., Matching Pennies, Rock-Paper-Scissors); mixed always does
  • Multiple Nash can coexist; not all are equally "good" or stable
  • Nash ≠ Pareto-optimal — Prisoner's Dilemma's (D,D) is Nash but both prefer (C,C)
  • Applies in multi-agent RL: stable joint policies are characterized by NE. Algorithms like Nash-Q (general-sum) and minimax-Q (zero-sum) compute equilibrium policies in stochastic / Markov games.
  • NOT an information condition: NE is a property of the strategy profile, not whether players observe each other. Bayesian NE handles incomplete-information games; standard NE doesn't require complete observation.
  • NOT restricted to deterministic games: NE concepts apply to stochastic games (Markov games), repeated games, and Bayesian games — NE is the unifying solution concept across strategic interactions.

Finding Pure-Strategy Nash (Best-Response / Underline Method)

For a 2-player normal-form game with cells (u₁, u₂): (1) For each COLUMN (each of P2's pure strategies), underline P1's best payoff in that column. (2) For each ROW (each of P1's pure strategies), underline P2's best payoff in that row. (3) Cells where BOTH numbers are underlined ⇒ pure-strategy Nash. Each underlined number = that player's best response to the other's column/row choice. Both underlined ⇒ mutual best response ⇒ Nash equilibrium.
  • 0 cells: no pure Nash; must compute mixed via indifference (below). E.g., Matching Pennies.
  • 1 cell: unique pure Nash. Often a dominant-strategy game (e.g., Prisoner's Dilemma's (D,D)).
  • 2+ cells: multiple pure Nash; usually one mixed Nash also exists "between" them (coordination, Battle of Sexes, chicken).

Mixed-Strategy Indifference Principle

In a mixed Nash, every pure strategy in the support of your mix yields the same expected payoff (otherwise you'd shift weight to the better one). Solve for the OPPONENT's mix by setting your own expected payoffs equal across your pure strategies.

2×2 recipe: P1 actions {T, B}; P2 actions {L, R}. Let q = P(P2 plays L) ⇒ solve for q by making P1 INDIFFERENT: E[P1 | T] = q · u₁(T,L) + (1−q) · u₁(T,R) E[P1 | B] = q · u₁(B,L) + (1−q) · u₁(B,R) Set equal, solve for q. (Symmetric: find p by making P2 indifferent across L vs R.)

Worked — Matching Pennies (zero-sum):

L R T (+1, −1) (−1, +1) B (−1, +1) (+1, −1) P1 indifferent ⇒ q·(+1) + (1−q)·(−1) = q·(−1) + (1−q)·(+1) 2q − 1 = 1 − 2q ⇒ q = 1/2. Symmetric ⇒ p = 1/2. Mixed Nash: both 50/50. Game value = 0.

Zero-Sum & Minimax

Zero-sum: total payoffs sum to 0 (one player's gain = other's loss). Strictly competitive.

Maximin (row player): max_i min_j u(i, j) ↑pick the row whose worst-case is least bad↑ Von Neumann theorem (1928): In any finite zero-sum game, max_i min_j = min_j max_i (with mixed strategies) This common value = the GAME VALUE
  • Mixed minimax can be worse than pure for a fixed mixture if the mix isn't the equilibrium one — randomizing 60/40 between two row choices weights toward the worse-case row, lowering the min. Only the equilibrium mix attains the game value.
  • Folk Theorem does NOT apply to (single-shot) zero-sum games — there's nothing to cooperate about

Stochastic (Markov) Games — multi-agent MDP

Setup: generalize the MDP to multiple players. A stochastic game is a tuple (S, A₁, ..., Aₙ, T, R₁, ..., Rₙ, γ) — same as MDP but each player has their own action set and their own reward function. At each state, all players act simultaneously; T(s, a₁, ..., aₙ, s′) gives the joint transition. n=1 ⇒ MDP; T trivial ⇒ matrix game.

The wedge: uniqueness of equilibrium value at each stage game. Zero-sum has it (Von Neumann minimax theorem) ⇒ Bellman backup is a contraction ⇒ everything works like single-agent RL. General-sum doesn't ⇒ central headache of MARL.

Mitigations for general-sum games (option d on Q21):

  • Side payments (transferable utility): players can compensate each other, aligning incentives. Turns Prisoner's-Dilemma-like games cooperative.
  • Repeated stochastic games: infinite-horizon + discount enables Folk-Theorem cooperation via credible punishment threats.
  • Communication / coordination protocols: pre-agreed conventions resolve equilibrium selection (e.g., always coordinate on row-player's preferred Nash in BotS-style stages).

Common traps: minimax-Q ≠ Nash-Q (different game classes — minimax-Q for zero-sum, Nash-Q for general-sum). VI does NOT generally converge in general-sum games. Q* is NOT unique in general-sum games.

4 · Repeated Games & the Folk Theorem

Repeated Game Setup

  • Same stage game played indefinitely; players observe history
  • Total payoff = discounted sum: Σ δᵗ · u_t with discount δ ∈ [0,1)
  • Strategies can be history-dependent ⇒ rich behaviors emerge that don't exist in one-shot games

The Folk Theorem (informal)

In an infinitely repeated game with sufficient discount factor, any feasible & individually rational payoff vector can be sustained as a Nash equilibrium via credible punishment for deviations.

  • Feasible: achievable as a (possibly mixed) average of stage payoffs
  • Individually rational: each player gets at least their minimax (worst-case-guaranteed) value
  • Mechanism: threat of future punishment outweighs the one-shot gain from deviating ⇒ cooperation sustainable
  • Requires infinite (or unknown) horizon — backward induction unravels cooperation in finite-known games
  • Requires high enough δ — if future doesn't matter, threats don't bite
  • Requires credible threats — see SPE below

Subgame Perfect Equilibrium (SPE)

Strategy profile is a Nash equilibrium in every subgame, including off-path histories. Stronger than Nash: rules out empty threats that wouldn't actually be carried out.

  • Solved via backward induction: start from the end of the game, find optimal play, propagate back
  • Not every Nash is SPE — non-credible punishment threats can support Nash but fail SPE
  • Requires punishment to be optimal even when triggered — else the deviator knows you won't follow through

Common Repeated-Game Strategies

Discount Factor Intuition

  • High δ (future matters): threat of future punishment is heavy ⇒ cooperation sustainable
  • Low δ (future doesn't matter): one-shot gain from defecting outweighs distant punishment ⇒ cooperation collapses
  • Quantitative threshold (Prisoner's Dilemma + Grim Trigger): cooperation sustainable iff δ ≥ (T − R) / (T − P), where T=temptation, R=reward, P=punishment payoffs

Common Game-Theory Pitfalls

  • Folk Theorem ≠ "any outcome works" — needs feasible, individually rational, infinite horizon, credible punishment
  • Nash ⇏ SPE: Nash can rely on threats that wouldn't be carried out; SPE requires credibility everywhere
  • Zero-sum ≠ all competitive games — most real interactions have positive-sum opportunities
  • Mixed strategy ≠ uncertain about choice — it's a deliberate randomization to make opponent indifferent
  • Heavy future discounting kills cooperation — players want their treats now

5 · Deep RL — DQN & Rainbow

DQN — Deep Q-Network (Mnih et al. 2015)

Q-learning with a neural net function approximator. Two stability tricks make it work:

  • Experience replay: store transitions (s, a, r, s′) in a buffer; sample minibatches uniformly to break temporal correlation
  • Target network: separate frozen copy of Q used for computing targets; sync to online net every k steps. Prevents the "moving target" instability of bootstrapping toward a network that's also being updated.
Target: y = r + γ · max_{a'} Q_target(s', a') Loss: L = E[(y − Q_online(s, a))²] ← TD loss on minibatch from replay

Rainbow DQN (Hessel et al. 2017) — six improvements stacked

Six independent ablations over baseline DQN, each addressing a different failure mode. Combined ≈ 2–3× median human-normalized score on Atari 57.

ComponentMechanismFailure mode addressed
Double DQN
(Hasselt 2015)		Decouple action selection (online net) from action evaluation (target net):
y = r + γ · Q_target(s', argmax_a Q_online(s', a))		Q-value overestimation from max(noisy estimates)
Prioritized Experience Replay
(Schaul 2015)		Sample transitions w.p. ∝ |TD-error|α; correct sampling bias with importance-sampling weights (1/(N·P(i)))βUniform replay wastes updates on already-learned transitions
Dueling Networks
(Wang 2016)		Two heads: V(s) and A(s, a); recombine as Q = V + A − mean_a' A(s, a').
Shared features feed both streams		Sample inefficiency when action choice is moot — V(s) learns even without trying every action
n-step Returns
(Sutton 1988)		Bootstrap target after n real steps:
R⁽ⁿ⁾_t = r_t + γr_{t+1} + ... + γⁿ⁻¹r_{t+n−1} + γⁿ·max_a Q(s_{t+n}, a)		1-step bootstrap propagates rewards too slowly; faster credit assignment (cost: higher variance)
Distributional RL (C51)
(Bellemare 2017)		Predict the distribution of returns Z(s,a) over 51 atoms in [V_min, V_max], not just the mean. Categorical projection of the distributional Bellman update; cross-entropy lossScalar Q-values discard higher-order info; capturing return distribution improves stability
NoisyNets
(Fortunato 2017)		Replace linear layers with y = (μ_W + σ_W ⊙ ε_W)·x + ...; σ is learned ⇒ state-dependent exploration noiseCrude exploration via fixed ε-schedule; agent learns when/where to explore
