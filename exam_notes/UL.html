CS 7641 — Unit 3: Unsupervised Learning  Quick Reference

1 · Clustering

Setup

Partition n unlabeled points into k groups of similar items. No ground truth — quality judged by internal metrics (silhouette, inertia, BIC) and downstream utility.

  • Standardize first: all distance/likelihood-based methods are scale-sensitive; large-magnitude features dominate. (GMMs are not immune — covariance still skews.)
  • Curse of dimensionality: distances concentrate in high-d; "nearest" loses meaning above ~20 dims; clustering quality degrades

K-Means — hard, spherical

Idea: minimize within-cluster sum-of-squares. Each point joins its nearest centroid; centroids re-fit to their members; repeat until stable.

Inertia (objective): J = Σᵢ ‖xᵢ − μ_{c(i)}‖² (LOWER = tighter clusters = better) Lloyd's algo: 1) Assign: c(i) ← argmin_k ‖xᵢ − μ_k‖ 2) Update: μ_k ← mean of points assigned to k Repeat until assignments stop changing
  • Assumptions: spherical (isotropic) clusters, similar sizes, Euclidean geometry
  • Fails on: non-convex shapes (crescents), elongated clusters, very different cluster sizes (centroid drifts toward the bigger one)
  • Init sensitivity: can land in local minima; k-means++ seeds centroids spread out (proportional to D²); use multiple restarts and keep the lowest-inertia run
  • Outlier sensitivity: distance is squared in the inertia objective and centroids are arithmetic means — a single far point can drag μ_k off the cluster body. Mitigations: trim outliers, use K-medoids (medoids are robust), or DBSCAN (handles noise natively).
  • Hard assignment — every point belongs to exactly one cluster, no uncertainty

Four things K-Means needs help with (memory hook): K (selection — elbow / silhouette), init (k-means++ + restarts), scale (standardize features), outliers (remove or switch to K-medoids). Anything else (dataset size, feature "unequal importance") is a distractor.

GMM with EM — soft, ellipsoidal

Generative story: (1) Pick cluster zᵢ = k with prior probability π_k; (2) Draw xᵢ ~ 𝒩(μ_k, Σ_k). Marginalizing out the hidden zᵢ gives the mixture density:

p(x | θ) = Σ_k π_k · 𝒩(x | μ_k, Σ_k) x = one observed data point (one row of data) θ = { π_1..π_K, μ_1..μ_K, Σ_1..Σ_K } (all params) π_k = P(z=k) = prior probability of cluster k (Σ_k π_k = 1) μ_k = mean of cluster k Σ_k = covariance of cluster k zᵢ = latent cluster index for xᵢ (unobserved)

Reading p(x | θ): weighted avg over clusters — each term = (cluster k's prior) × (likelihood of x under cluster k). Sum because we don't know which cluster generated x. The hidden zᵢ is the latent variable EM is built to handle.

Goal: find θ that maximizes the log-likelihood log L(θ) = Σᵢ log p(xᵢ | θ). Direct optimization is hard because of the unobserved zᵢ ⇒ EM alternates soft assignments and parameter refits until log L(θ) stabilizes.

E-step (responsibilities = posterior over cluster): γ_ik = P(zᵢ=k | xᵢ, θ) = π_k · 𝒩(xᵢ; μ_k, Σ_k) / Σⱼ π_j · 𝒩(xᵢ; μ_j, Σ_j) M-step (refit params using responsibilities as weights): N_k = Σᵢ γ_ik π_k = N_k / n μ_k = (1/N_k) · Σᵢ γ_ik · xᵢ Σ_k = (1/N_k) · Σᵢ γ_ik · (xᵢ − μ_k)(xᵢ − μ_k)ᵀ
  • Three knobs per cluster: μ_k (location), Σ_k (shape/orientation), π_k (size/prior)
  • Handles unequal sizes & shapes via per-component π_k and Σ_k
  • Soft assignments capture uncertainty in overlap regions
  • EM is coordinate ascent on a lower bound to log-likelihood
  • Local optima: re-run from different inits; log-likelihood rises per iteration, finals differ
  • More k always raises training log-likelihood ⇒ use BIC/AIC to choose k

K-Means = EM on GMM (degenerate limit)

Three constraints collapse the GMM EM machinery into Lloyd's algorithm:

ConstraintWhat it kills
π_k = 1/K  (equal mixing weights, fixed)Cluster size prior
Σ_k = σ²I for all k  (shared spherical, fixed)Cluster shape/orientation — only Euclidean distance matters
σ² → 0  (conceptual limit)Softmax responsibilities sharpen into hard 0/1 assignments

K-Means itself has no σ² parameter — the σ² → 0 limit is the analytical bridge that turns GMM's soft responsibilities into K-Means' hard nearest-centroid choice.

E-step under these constraints:

γ_ik = exp(−‖xᵢ − μ_k‖² / 2σ²) / Σⱼ exp(−‖xᵢ − μⱼ‖² / 2σ²) └────────── softmax over distance, sharpens as σ² → 0 ──────────┘ σ² → 0: γ_ik → 1 if k = argmin_j ‖xᵢ − μⱼ‖, else 0 ← K-Means assignment

M-step — same formula as GMM EM, just evaluated with hard γ_ik ∈ {0, 1}:

μ_k = (1/N_k) · Σᵢ γ_ik · xᵢ = (1/N_k) · Σ_{i: assigned to k} xᵢ ← only assigned points contribute (γ_ik=1) = mean of points in cluster k ← Lloyd's centroid update

K-Means throws away GMM's machinery for shape (Σ_k), size (π_k), and uncertainty (soft γ). Forced spherical + equal-prior + zero-variance ⇒ EM degenerates into hard nearest-centroid + mean-update = Lloyd's algorithm.

Hierarchical / Linkage Clustering

Idea: agglomerative (bottom-up): start with each point as its own cluster; repeatedly merge the closest two until one cluster remains. The output is a dendrogram — cut at any height for any k.

Linkage rule = how we define the distance between two existing clusters A and B (used to decide which pair to merge next). All linkages below use pairwise distances between original points, not cluster means.

LinkageDistance between A and B =Tendency
Singlemin over all (a, b) with a ∈ A, b ∈ B — i.e., the closest cross-cluster pairChains — one bridging pair is enough to merge; long stringy clusters; sensitive to noise
Completemax over all (a, b) — i.e., the farthest cross-cluster pairCompact, ball-like; clusters merge only when fully tight; sensitive to outliers
Averagemean of all cross-cluster pair distancesBalanced; robust default
Wardincrease in within-cluster variance after the merge (uses centroids)Like K-Means but hierarchical — favors compact, equal-variance clusters
  • Single linkage chaining is the canonical failure mode: a thin bridge of points merges two distinct clusters into one. Bad in noisy / high-d settings.
  • O(n²) memory for the distance matrix ⇒ doesn't scale to very large n

DBSCAN — density-based

Idea: a cluster is a region of high density; points outside any dense region are noise. Two hyperparams: ε (neighborhood radius), minPts (density threshold).

  • Core point: ≥ minPts within ε. Cluster grows by chaining core points; border points attach to a cluster but don't extend it; noise points remain unassigned.
  • No need to specify k; finds arbitrary shapes (crescents, rings); explicitly labels noise
  • Fails when clusters have varying density (single ε can't fit both); degrades in high-d (curse of dimensionality breaks ε neighborhoods)

Choosing k

  • Elbow method (K-Means): plot inertia vs k. Inertia always drops as k grows (more centroids → smaller distances, k=N gives J=0); pick the "kink" where the drop levels off.
  • Silhouette score: per-point (b−a)/max(a,b) where a = mean intra-cluster dist, b = mean dist to nearest other cluster. Range [−1, 1], higher is better (≈1 well-clustered, ≈0 on boundary, <0 likely wrong cluster).
  • BIC / AIC (GMM): penalize log-likelihood by # parameters; lower is better. BIC penalty (k·log n) is stricter than AIC's (2k) ⇒ BIC favors simpler models.
  • Optimizing in-sample fit alone (= minimizing inertia for K-Means or maximizing log-likelihood for GMM) always rewards larger k ⇒ over-segmentation. Practical pairings: K-Means → elbow + silhouette (elbow = diminishing returns; silhouette = geometric quality); GMM → BIC/AIC (built-in complexity penalty; silhouette assumes spherical clusters so it under-rates GMM's ellipsoidal fits).

Quick decision rules:

  • Curved / non-convex boundaries → single-link or DBSCAN (DBSCAN if noise; single-link if clean)
  • Noise / outlier label visible → DBSCAN (only method that produces one)
  • Smooth elliptical soft regions → EM (GMM)
  • Overlapping clusters → EM only (others can't represent overlap)
  • Straight Voronoi-style cuts between ball-like blobs → K-Means / Ward / complete-link
  • Cluster sizes very unequal → EM (others bias toward balance)

2 · Feature Selection

Why select features?

  • Generalization: high-d, low-n (d ≫ n, e.g. genomics) ⇒ models fit noise. Removing irrelevant features reduces overfitting.
  • Interpretability & cost: fewer features = simpler model, cheaper inference, easier to explain
  • Selection ≠ transformation: selection picks a subset of original features; PCA/ICA create new ones

Three Families

FamilyHowProsCons
FilterScore each feature individually (correlation, MI, χ², ANOVA F); rank; keep top-kFast · scales to 1000s of features · model-agnosticIgnores interactions; redundant top-k
WrapperSearch subsets; evaluate each via CV with the actual model (forward, backward, RFE)Captures interactions & redundancy; tailored to modelExpensive (exponential subsets); overfits in low-n; needs nested CV
EmbeddedSelection happens during model training (Lasso, tree feature importance, regularized SVM)Model-aware & cheap; principledTied to one model class

Relevance vs Usefulness

Relevance: "does this feature carry information about y?" (statistical property of the feature)
Usefulness: "does this feature improve this model's performance?" (depends on model + other features)

  • Relevant but useless: a feature can be highly correlated with y yet redundant given another feature already in the model
  • Irrelevant individually but useful jointly: features can be uninformative alone yet predictive in combination — XOR is the canonical example. Trees/NNs surface these; filter methods miss them.
  • ⇒ Filter methods rank by relevance; wrappers measure usefulness directly

Practical Pitfalls

  • Selection on training set leaks: wrapper-style search on the same data used for evaluation overfits ⇒ nested CV (outer loop for evaluation, inner loop for selection)
  • One-hot explosion: dozens of sparse one-hots fragment clusters and add noise — prefer parsimonious encodings or embeddings
  • Wrappers don't guarantee global optimum: forward/backward search is greedy; combinatorial space too large to enumerate

Real Challenges of Feature Selection

  • Bias from removing important features: dropping a relevant feature loses information ⇒ higher bias. Selection trades variance reduction for potential bias increase — that's the bias-variance tradeoff applied to features.
  • Joint relevance (XOR trap): a feature can be uninformative alone yet essential in combination — filter methods miss this entirely.
  • Computational cost: wrapper subset search scales as 2ᵈ exhaustively, O(d²) greedily. Filter methods are cheap; embedded methods (Lasso) intermediate.
  • Simplicity vs informativeness balance: too few features ⇒ underfit; too many ⇒ overfit + interpretability/cost issues. Goal is the sweet spot, not "remove as many as possible."
  • NOT supervised-only: unsupervised feature selection exists (variance threshold, Laplacian score, autoencoder-based, redundancy removal). Selection is a preprocessing tool, not a label-dependent one.
  • NO accuracy guarantee: selection can help (remove noise), hurt (drop relevant features), or do nothing. Cross-validate the selection itself; "always improves accuracy" is false.

3 · Feature Transformation

PCA — variance via orthogonal axes

Idea: find orthogonal directions of maximum variance. Project data onto top-k of these directions ⇒ best linear k-D approximation in least-squares sense.

1) Center data: X̃ = X − x̄ 2) Covariance: C = (1/n)·X̃ᵀX̃ (d × d) 3) Eigendecompose: C = V Λ Vᵀ Eigenvectors v_k → principal axes (orthogonal) Eigenvalues λ_k → variance along axis k 4) Sort columns of V by DESCENDING λ_k ← largest variance first = PC1 5) Project: Z = X̃·V_{:,1:k} (n × k) Variance explained by component k: λ_k / Σⱼ λⱼ Equivalently: SVD of X̃ — same answer, more numerically stable, sorts for free
  • What it does: decorrelates features (Cov(Z) is diagonal); compresses by keeping top-variance directions
  • Picking k: "knee" in scree plot, or smallest k with cumulative variance ≥ 90/95%
  • Best when features are collinear — strong linear redundancy collapses into a few high-λ directions
  • High variance ≠ informative: top components can capture noise, batch effects, or irrelevant dominant axes; not guaranteed to help downstream tasks
  • Overfits in d ≫ n regime — sample covariance is rank-deficient; some "signal" directions are noise
  • Standardize first if features have different units; otherwise high-magnitude features dominate
  • Components are linear combinations of all original features ⇒ not interpretable as original variables

Drawing PC1/PC2 by inspection (2D)

  • PC1 = direction of max variance — eyeball as the long axis of the data cloud
  • PC2 = orthogonal to PC1 with max remaining variance (in 2D: simply perpendicular)
  • Both axes pass through the centroid; arrow length ∝ √λ_k = std dev along that axis
  • Why max variance = long axis: variance along unit direction u is uᵀCu (a quadratic form in the covariance matrix). Maximizing over unit u → top eigenvector of C = PC1; second max subject to orthogonality → PC2.

Exam gotchas:

  • Circular cloud: "PCA is degenerate; eigenvalues equal so PC1/PC2 are arbitrary" is the correct answer — don't try to invent a direction.
  • Cross / V-shape: PC1 aligns with the dominant arm, missing the cross entirely. Flag PCA's linear limitation; kernel PCA or manifold methods are the fix.
  • Outliers drag PC1: PCA minimizes squared error, so far points have outsized influence on the principal direction.

ICA — independence via non-Gaussianity

Idea: assume observed data is a linear mixture of statistically independent latent sources; recover the unmixing matrix W. Classic cocktail party problem: separate two voices from two microphones.

Model: x = A·s (x observed, s independent sources, A mixing matrix) Recover: s ≈ W·x by maximizing non-Gaussianity of W·x Common objectives: · kurtosis (proxy for non-Gaussianity, sensitive to outliers) · negentropy J(y) = H(y_Gauss) − H(y) (preferred, robust) · maximum likelihood with non-Gaussian density
  • Why non-Gaussianity? Central Limit Thm: sum of independent rvs is "more Gaussian" than its parts. So the most non-Gaussian projection is the most "un-mixed."
  • Gaussian sources are unrecoverable: ICA needs at most one Gaussian source; if all sources are Gaussian, mixing is non-identifiable (rotations are equivalent)
  • Indeterminacies: sources recovered up to (i) permutation and (ii) scale/sign — order and amplitude are not fixed
  • Data-hungry & unstable: estimating higher-order stats needs more samples than PCA; collinearity hurts; can extract spurious components if true sources are near-Gaussian

Common traps: ICA finds independent axes, NOT "linearly dependent" axes (word-swap trap). Both PCA and ICA are unsupervised — neither uses labels. ICA does NOT maximize variance — it whitens variance away first, then maximizes non-Gaussianity.

Uncorrelated ≠ independent. Uncorrelated means only the linear (2nd-order) cross-moment vanishes: Cov(X, Y) = E[XY] − E[X]·E[Y] = 0. Independence is strictly stronger — the joint factorizes p(X, Y) = p(X)·p(Y), which forces E[f(X)·g(Y)] = E[f(X)]·E[g(Y)] for every pair of functions f, g (i.e. all cross-moments factor, not just the linear one).

Independent ⇒ uncorrelated (always). The reverse only holds when the joint is Gaussian (where mean & covariance fully describe the distribution).

Classic counter-example: X ~ Uniform(−1, 1), Y = X² E[X] = 0, E[XY] = E[X³] = 0 ⇒ Cov(X, Y) = 0 (uncorrelated) but Y is a deterministic function of X (NOT independent)

This is why PCA is enough for Gaussian data (all dependence is 2nd-order ⇒ decorrelating = independence) but ICA must look at higher-order statistics for non-Gaussian sources — and conversely why ICA cannot separate Gaussian sources (no higher-order signal to exploit; see ICA warning above).

4 · Manifold Learning

The Manifold Hypothesis

Real high-d data often lies on a low-d manifold (curved surface) embedded in ambient space. Linear methods (PCA) flatten it incorrectly; manifold learners aim to unroll the surface.

Canonical example: the Swiss roll — 2D surface curled in 3D. PCA squashes it onto its longest axis (wrong); Isomap unrolls it.

MDS & Sammon Mapping (distance preservation)

Classical MDS: preserves all pairwise Euclidean distances Stress = Σᵢⱼ (d*ᵢⱼ − dᵢⱼ)² (unweighted) ≡ PCA when input distances are Euclidean Sammon: weighted stress, weights ∝ 1/d*ᵢⱼ Stress_S = Σᵢⱼ (d*ᵢⱼ − dᵢⱼ)² / d*ᵢⱼ → emphasizes SMALL original distances
  • Sammon deliberately overweights short distances ⇒ preserves local structure better than plain MDS, reducing crowding
  • Both are non-parametric: no inductive map for new points without re-running

Isomap — geodesics on a kNN graph

  1. Build kNN graph on data points
  2. Estimate geodesic distance between any two points = shortest path on the graph
  3. Run classical MDS on the geodesic distance matrix ⇒ embedding
  • Unrolls curved manifolds (Swiss roll, "A" digits with angle/size variation) — geodesic ≈ true intrinsic distance, not ambient Euclidean
  • Short-circuit failure: if k is too large, the kNN graph creates spurious edges across folds of the manifold ⇒ geodesics become straight-line shortcuts ⇒ embedding corrupted. Tune k carefully.

Laplacian Eigenmaps — local neighborhoods

Idea: build a weighted kNN graph; embed so that strongly-connected points stay close. Solves the generalized eigenproblem on the graph Laplacian:

W: weighted kNN graph (e.g. Gaussian heat kernel on edges) D: diagonal degree matrix (Dᵢᵢ = Σⱼ Wᵢⱼ) L = D − W (graph Laplacian) Solve: L f = λ D f Take the smallest non-trivial eigenvectors as embedding coords.
  • Smallest eigenvectors = smoothest functions on the graph ⇒ preserves local adjacency
  • Trivial λ=0 eigenvector (constant) is dropped — it carries no info
  • Closely related to spectral clustering (same Laplacian, different downstream use)

t-SNE — local affinity, KL objective

Goal: visualize high-d data in 2D such that local structure (neighborhoods, clusters) is preserved — even at the cost of distorting global geometry. Best EDA tool for asking "are there cluster-like groups in my data?"

Three core tricks:

  1. Distances → probabilities (SNE): raw high-d distances are noisy, so model "probability that j is a neighbor of i" via a Gaussian kernel anchored at xᵢ (i indexes data points, not features). σᵢ is the std dev of that kernel — one scalar per point, isotropic across all d feature directions. It's tuned per-point so each point sees the same effective neighborhood size = perplexity (small σᵢ in dense regions, large σᵢ in sparse ones — adapts to local density).
  2. Asymmetric KL ⇒ local emphasis: minimizing KL(P ‖ Q) heavily penalizes "false friends" (far high-d points placed close in 2D) but cheaply forgives "missed neighbors" (close points placed apart). ⇒ aggressive preservation of local neighborhoods, willing sacrifice of global geometry.
  3. Student-t in low-d (the "t" of t-SNE) fixes crowding: high-d volume grows faster than 2D, so a Gaussian-Gaussian version (original SNE) crowds moderate-distance points in the middle. Heavy-tailed Student-t lets moderate distances spread out ⇒ visible cluster gaps.
High-d affinities P (Gaussian kernel; perplexity sets bandwidth): pⱼ|ᵢ ∝ exp(−‖xᵢ − xⱼ‖² / 2σᵢ²) pᵢⱼ = (pⱼ|ᵢ + pᵢ|ⱼ) / (2n) Low-d affinities Q (Student-t kernel, heavy tails ⇒ less crowding): qᵢⱼ ∝ (1 + ‖yᵢ − yⱼ‖²)⁻¹ Optimize: KL( P ‖ Q ) via gradient descent on {yᵢ}
  • Perplexity ≈ effective neighborhood size (typical 5–50); small ⇒ very local, large ⇒ broader
  • Early exaggeration: scale up P early in optimization ⇒ tighter clusters with bigger gaps
  • Distorts global geometry: inter-cluster distances and cluster areas are NOT meaningful — blob size ≠ class prevalence
  • No native out-of-sample mapping (must re-run with new points); stochastic across runs

UMAP — fuzzy simplicial sets, cross-entropy

Goal: same as t-SNE — visualize HD data in 2D, preserve neighborhoods, reveal clusters. Different machinery; broadly similar empirical output, with somewhat better global structure, faster runtime, and an out-of-sample transform.

The intuition in one line: build a weighted graph on your HD data where edge weights = "how confident am I that these two points are neighbors," then lay it out in 2D as a force-directed graph (springs pull connected points together, random pairs repel) until the 2D graph "matches" the HD graph.

Four core tricks (mirroring t-SNE's three):

  1. HD graph as a fuzzy simplicial set: for each point xᵢ, its k nearest neighbors form a fuzzy neighborhood with edge weights wᵢⱼ ∈ [0, 1] — close = strong (≈1), far = weak (≈0). "Fuzzy simplicial set" is just fancy topology-speak for this weighted kNN graph. This is UMAP's analog of t-SNE's HD probability matrix P.
  2. Adaptive bandwidth (analog of perplexity): per-point parameters ρᵢ (distance to nearest neighbor) and σᵢ (scale) are set so each xᵢ ends up with ≈ log₂(n_neighbors) effective connections. Dense regions ⇒ small σᵢ; sparse regions ⇒ large σᵢ. Same idea as t-SNE's per-point Gaussian σᵢ, just with a different formula.
  3. Symmetrize via fuzzy union (probabilistic OR): the raw graph is asymmetric (i may consider j a neighbor but not vice versa). UMAP uses w(i,j) = wᵢⱼ + wⱼᵢ − wᵢⱼ·wⱼᵢ — "either direction counts" — instead of t-SNE's average. Result: a single symmetric fuzzy graph in HD.
  4. Cross-entropy loss + force-directed SGD: in 2D, edge weights vᵢⱼ come from a Student-t-like kernel. Minimize cross-entropy CE(w ‖ v) — equivalent to two forces:
    • Attractive on each HD edge: pull yᵢ and yⱼ together with strength ∝ wᵢⱼ
    • Repulsive on random non-edges (negative sampling, ~5 per positive): push them apart
    Optimize via stochastic edge-sampling SGD — no full pairwise gradient ⇒ linear-time per epoch, which is why UMAP scales.
HD fuzzy weight (i → j): wᵢⱼ = exp( −max(0, dᵢⱼ − ρᵢ) / σᵢ ) ─── ── ── dist near scale nbr (set so Σⱼ wᵢⱼ ≈ log₂(n_neighbors)) Symmetrize: w(i,j) = wᵢⱼ + wⱼᵢ − wᵢⱼ·wⱼᵢ (fuzzy OR) LD weight (Student-t-like; min_dist sets the spacing): vᵢⱼ = (1 + a · ‖yᵢ − yⱼ‖²ᵇ)⁻¹ Loss: CE(w ‖ v) = Σ_{ij} [ w log(w/v) + (1−w) log((1−w)/(1−v)) ] ───────────────── ───────────────────────────── attractive term repulsive term (HD edges pull) (non-edges push, via neg. sampling)

Why this gives better global structure than t-SNE: the cross-entropy loss penalizes BOTH "false friends" (close in 2D, far in HD) and "missed neighbors" (far in 2D, close in HD) — symmetric, unlike KL(P‖Q) which only punishes the first. So UMAP is less willing to crush long-range structure to perfect locals.

  • n_neighbors: like t-SNE perplexity — small=local, large=more global. min_dist: minimum spacing in 2D; smaller ⇒ tighter clusters with bigger gaps.
  • Generally better global structure than t-SNE; faster on large data (edge-sampling SGD); supports inductive transform (place new points without re-fitting); has a supervised mode
  • Spectral init by default ⇒ more reproducible than t-SNE's random init
  • No early exaggeration phase (that's a t-SNE thing)
  • Both methods are scale-sensitive (Euclidean distances in HD). Standardize features first — but neither method requires a specific range like [0,1]. "t-SNE is scale-invariant" is false.

When NOT to use t-SNE / UMAP

  • kNN retrieval / quantitative distances: 2D distances aren't faithful to high-d distances
  • Stable production embeddings: stochastic outputs, hyperparameter-fragile, hard to monitor
  • Out-of-sample mapping: vanilla t-SNE has none; UMAP's transform isn't metric-preserving
  • Variance interpretation: axes have no scale meaning — area ≠ prevalence, gap ≠ dissimilarity
  • Batch-effect correction: neither method does this — embeddings can reflect cohort artefacts
  • Use them for: exploratory visualization, finding cluster candidates, qualitative inspection
